Lesson 0003
Inner Loop

Reading time: ~20 minutes. No coding required.

In the previous lesson, you learned about the harness — the scaffolding around agents. Now let's go inside the agent itself. The inner loop is the process by which a single agent completes a single task. This is the domain of the Agent Engineer.

The Inner Loop: Agent in Action

Every time an agent works on a task, it goes through a cycle:

Think → Plan → Act → Observe → Think

The loop continues until the agent decides the task is complete. This cycle happens inside a single session, within a single context window.

Four Things That Make an Inner Loop Work Well

1. Clear Objective

The agent must know exactly what "done" looks like. Vague instructions produce vague results. A good objective includes: what to produce, where to save it, what format to use, and how to verify it.

2. Good Tools

The agent needs the right tools for the job. A coding agent needs file access and a terminal. A research agent needs web search. A writing agent needs document formatting tools. Without the right tools, the agent cannot act effectively.

3. Context That Matters

Feed the agent only what it needs to make good decisions. Irrelevant context is noise. The agent should not have to sift through 50 files to find the 2 that matter.

4. Self-Correction Triggers

The agent needs checkpoints — moments where it asks: "Am I on the right track?" Without self-correction, the agent can go very far down the wrong path before anyone notices.

Prompt Optimization: Making Instructions Work

A prompt is not just "do X." A good harness prompt for an inner loop task includes:

  • Role — Who is the agent? (e.g., "You are a security vulnerability researcher.")
  • Task — What specifically to do? (e.g., "Research CVE-2024-XXXX and write a summary.")
  • Output format — What should the result look like? (e.g., "A markdown file in signals/cve-daily.md using the schema in signals/README.md.")
  • Constraints — What to avoid? (e.g., "Do not include speculation. Only cite official sources.")
  • Exit criteria — How does the agent know it is done? (e.g., "The summary covers: description, affected versions, severity, and remediation.")
Pattern. The best inner loop prompts are written like instructions for a new hire on their first day — specific, structured, and leaving no ambiguity about what success looks like.

Context Injection: What to Feed the Agent

There are 3 ways to give an agent context:

System Prompt

Permanent instructions baked into the agent's configuration. Use for role definition, behavioral rules, and output format. Keep it stable — it should not change per task.

Task Prompt

Instructions given at the start of each task. Use for task-specific goals, constraints, and success criteria. This is the variable part — different tasks, different prompts.

Dynamic Context

Information injected during the loop as the agent works. Use for file contents, search results, tool outputs. This is real-time — it changes as the agent gathers information.

Common Inner Loop Failure Modes

Agents fail in predictable ways. Knowing these helps you design better harnesses:

Failure Mode Symptom Fix
Goal drift Agent starts doing something unrelated to the task Add explicit checkpoint prompts: "Are you still on track? Review the objective."
Context overflow Agent gets confused or produces generic output Reduce context. Feed only the top 3 most relevant files.
Infinite loop Agent calls the same tool repeatedly without progress Add a step counter or max-iterations guard in the harness.
Hallucination Agent cites facts or sources that do not exist Require verifiable sources. Add a tool that confirms facts, not just retrieves them.
Tool misuse Agent uses the wrong tool for the job Make tool descriptions more explicit. Add examples of correct tool use.

Tool Selection: Choosing the Right Action

A core part of the inner loop is deciding which tool to use. This is called tool selection. A well-designed tool set makes this easy:

  • Distinct names — Each tool name clearly describes what it does. "search_web" not "tool_1."
  • Clear descriptions — Every tool has a description that tells the agent when to use it.
  • Explicit inputs/outputs — The agent knows exactly what to pass in and what to expect back.
  • Scoped purpose — Each tool does one thing well. Avoid "do everything" tools.

Key Takeaways

  • The inner loop is the Think → Plan → Act → Observe cycle a single agent goes through to complete one task.
  • Four things make inner loops work: clear objectives, good tools, relevant context, and self-correction triggers.
  • A good task prompt includes: role, task, output format, constraints, and exit criteria.
  • Common failure modes: goal drift, context overflow, infinite loop, hallucination, tool misuse.
  • The fix for most inner loop problems is better prompts and harness design, not a smarter model.

Questions to Think About

  1. Think of a task you do repeatedly at work. What would a perfect inner loop prompt look like for that task? (Role? Task? Output format? Constraints?)
  2. Have you seen an agent fail in one of these failure modes? Which one? What was the fix?
  3. What is the "right tool" for a security consultant researching a new CVE? What does that tool need to do?
If anything is unclear, ask. The inner loop is the foundation of all agent work. Before moving to the outer loop, make sure you can spot failure modes and write a good task prompt.
