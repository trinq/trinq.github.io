# Mission — The 4-Loop Ladder

## Why this course exists

You're using AI agents. They're useful. They also stall out — the work you hand them gets half-done, the output is "almost right," and you spend your evening patching it up.

There is a **ladder of four loops** that turns "an agent you poke" into "an agent that gets better while you sleep." Every rung adds a layer of automation around the previous one. Most people stop at rung 2. Almost nobody is on rung 4 yet — and that's where the leverage is.

## The destination

By the end of this course, you will be able to:

1. **Name** all four loops and what each one does to the agent's output quality.
2. **Explain** why most people plateau at loop 2 and what unlocks loops 3 and 4.
3. **Identify** which loop you are currently on for any given agent in your stack.
4. **Pick** the next loop to climb based on a real task you already have.

## The math behind the pitch

If an agent improves by 1% every night, in a year it becomes **37.8× better** (1.01³⁶⁵ ≈ 37.78). That's the hill-climbing loop (Loop 4) compounding on top of everything below it. The other three loops give the agent the surface area it needs to keep improving.

## Who this course is for

Engineers who already build with agents — Cursor, Claude Code, LangChain, or any other framework. You don't need to be deep into one tool. You need to be tired of babysitting prompts.

## What this course is not

- Not a tutorial on LangChain. The four primitives (create_agent, RubricMiddleware, LangSmith Deployment, LangSmith Engine) are mentioned because they are the canonical names, but the ladder generalizes to any agent framework.
- Not a course on prompt engineering. Prompt engineering is now a layer inside Loop 1. The interesting skill is what wraps around the prompt.
