Glossary

Reference for the four-loop ladder. Each lesson references these terms. Use this as your one-stop lookup.

The four loops

Loop 1 — Agent loop
The model picks a tool, calls it, reads the result, picks another, and continues until it decides the task is done. The floor of the ladder. Canonical primitive: create_agent (LangChain).
Loop 2 — Verification loop
After Loop 1 finishes, a grader scores the output against a rubric. If it fails, feedback goes back to the agent and it retries, up to a cap. Canonical primitive: RubricMiddleware (deepagents).
Loop 3 — Event-driven loop
The agent runs because something fired (cron, webhook, Slack message, DB row), not because a human invoked it. The defining property: no human in the trigger path. Canonical primitives: LangSmith Deployment (cron / webhooks) or Fleet channels.
Loop 4 — Hill-climbing loop
An analysis agent reads traces from Loop 3 runs, clusters failures, proposes patches (prompt edits, tool changes, new rubric criteria), and applies them. The agent improves itself. Canonical primitive: LangSmith Engine.

Primitives and patterns

create_agent
The LangChain entry point for Loop 1. Pick a model, plug in tools, get back an invokable agent. The pattern repeats across frameworks.
Rubric
A list of criteria the grader checks against. Each criterion is one thing, testable, and either deterministic (checked by code) or fuzzy (checked by an LLM-as-judge). Good rubrics have 3-8 criteria.
RubricMiddleware
The deepagents middleware that attaches a grader to Loop 1. Takes a model (the grader, often smaller), an optional system prompt, optional tools the grader can call, and a max_iterations cap. Default cap: 3. Hard cap: 20.
LLM-as-judge
Using one LLM to score another LLM's output. Used for fuzzy checks (did it answer the question? is the tone right?). Slower and more expensive than deterministic checks but the only way to grade non-deterministic output.
Deterministic check
A pass/fail check that runs as code, not as an LLM. "Did the link resolve?" "Does the JSON match the schema?" "Is the file written?" Cheap, fast, unambiguous.
Trigger
The event that fires Loop 3. A cron schedule, an HTTP webhook, a Slack message, a database row, a GitHub issue event. The defining property: no human invocation.
Trace
The complete record of one agent run: system prompt, inputs, every tool call and result, the rubric verdict, the final output. Loop 4 reads traces.
Patch
Loop 4's unit of work. A small edit to the agent's setup — usually a sentence in the system prompt, a new tool, a new rubric criterion, or a model change. Patches are tested against failing traces before being kept.
Engine (LangSmith Engine)
The LangChain agent that automates Loop 4. Built on Deep Agents. Pulls traces, clusters failures into issues, proposes patches, verifies them.
Fleet channels
LangChain's chat-style triggers (Slack, Teams, etc.). Part of Loop 3's infrastructure.
Harness
The thing wrapping the agent: prompt, tools, rubric, triggers, trace storage, analysis agent. The harness is what improves over time on Loop 4. "The model hasn't been the bottleneck for months. What's left is the harness around it."
Hill-climbing
The metaphor for Loop 4. The agent keeps walking up the slope of its own failures — small improvements, applied iteratively, each one measured against the rubric.

The math

1.01365 ≈ 37.78
The geometric compounding formula. 1% improvement per day, compounded for a year, gives ~37.8× the starting value. Linear growth (1 + 0.01 × 365) gives only ~4.65. The difference between linear and compounding is the whole pitch of Loop 4.

Status vocabulary

satisfied
RubricMiddleware status: the grader accepted the output. The run terminates successfully.
needs_revision
RubricMiddleware status: the grader found at least one unmet criterion. The middleware sends feedback back to the agent for another attempt.
max_iterations_reached
RubricMiddleware status: the agent exhausted its retry budget without satisfying the rubric. The run terminates as a failure. Tweak the rubric, the prompt, or raise the cap.
failed
RubricMiddleware status: the grader itself errored. Different from max_iterations_reached — this is a grader problem, not an agent problem.
