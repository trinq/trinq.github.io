The Loop Ladder Cheat Sheet

Print this. Tape it above your monitor. The four rungs in one page.

The four rungs

Rung What it does What it costs Canonical primitive
Loop 1 — Agent Model calls tools until task is done 1× tokens create_agent
Loop 2 — Verification Grader scores output against rubric, retries on fail 2-3× tokens RubricMiddleware
Loop 3 — Event-driven Agent runs on triggers (cron, webhook, Slack) infra + 2-3× per run LangSmith Deployment / Fleet
Loop 4 — Hill-climbing Trace analysis agent patches the agent's setup analysis compute LangSmith Engine

Where you probably are

Rung Symptoms Next step
Loop 1 only You copy-paste agent output and patch it by hand Write a rubric. Run Loop 2.
Loop 2 You invoke the agent. It grades and retries. You read the result. Add a cron or webhook. Move to Loop 3.
Loop 3 Agent fires on events. You read its output in Slack / logs. Let traces accumulate. Start clustering failures. Move to Loop 4.
Loop 4 Agent patches itself. You see commits in the morning. Compound. Stay on this rung for a year.

The math

1.01365 ≈ 37.78

1% improvement every night, compounded for a year. That's the pitch of Loop 4 in one formula.

The arrow that matters

Loop 4
hill-climb ↑ patches Loop 1
agent runs → Loop 2
grade + retry → Loop 3
trigger

Loop 4 doesn't restart the agent. It reaches inside and edits the prompt, the tools, or the rubric. The system prompt tomorrow is not the system prompt today — because the agent itself edited it.

One-sentence summary of each loop

  • Loop 1: the agent runs.
  • Loop 2: the agent grades itself and retries when it fails.
  • Loop 3: the agent runs on its own, fired by events.
  • Loop 4: the agent improves itself, fired by its own failures.
