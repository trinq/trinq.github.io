Lesson 0006: Measure Your Impact

Biến lý thuyết thành số — đo thật trên task thật

Tại sao cần đo?

Không có số, bạn không biết ponytail có hoạt động không. Mission nói "ít code hơn, ít cost hơn, nhanh hơn" — nhưng con số nào?

Impact Formula

Đo impact bằng 3 metrics cơ bản:

-30%
Lines of Code saved
-20%
API calls saved
+50%
Rungs that fired

Chuẩn bị

Trước khi bắt đầu, chọn một task từ work thật của bạn:

  • Task đã có PRD hoặc issue rõ ràng
  • Task không quá lớn (ước tính < 100 lines code)
  • Task có thể implement trong 1-2 giờ

Task mẫu

"Thêm notification badge vào avatar user trong header" — đủ nhỏ để đo, đủ thật để học.

Workflow đo impact

Step 1 — Baseline (không ponytail)

Chạy agent mà không bật ponytail. Ghi lại trước:

Metric Before After
Lines of code ?
Files created ?
Dependencies added ?
Time elapsed ?

Quan trọng

Sau khi agent propose, không accept ngay. Ghi lại code proposal, rồi mới accept.

Step 2 — Với Ponytail

Reset lại codebase về trạng thái ban đầu, rồi chạy lại với ponytail:

git checkout -- .
# Hoặc git stash nếu đã apply code

Bật ponytail và tracking:

/ponytail full
/track start

Implement cùng task. Mỗi lần agent propose code:

  • Check rung nào fired
  • Ghi lại code bị reject
  • Ghi lại code được accept

Step 3 — So sánh

Metric Without Ponytail With Ponytail Savings
Lines of code ~80 ~45 -35 lines (-44%)
Files created 3 1 -2 files (-67%)
Dependencies 2 packages 0 -2 packages
Rungs fired — 4 reuse, stdlib, inline, minimum

Rungs thường fire nhất

Rung Tên Khi nào fire Impact
R2 Reuse Code đã tồn tại Highest
R3 stdlib Thư viện mới High
R4 native Platform API High
R5 one-liner Function đơn giản Medium
R6 minimum Core logic thừa Medium
R7 delete Code không cần Low (thường bỏ qua)

Bài tập

Hoàn thành lesson

Hoàn thành impact tracking cho 3 tasks:

  1. 1 bug fix nhỏ
  2. 1 feature nhỏ (< 50 lines)
  3. 1 feature vừa (50-100 lines)

Sau 3 tasks, bạn sẽ có baseline thật để so sánh.

FAQ

Q: Không có thời gian đo từng task?

Đo nhanh: chỉ cần lines of code + rungs fired. Không cần precision, chỉ cần direction.

Q: Task đầu tiên có thể không tiết kiệm gì?

Bình thường. Agent còn học. Sau 3-5 tasks, pattern sẽ rõ.

Q: Vẫn cần TDD?

Có. Ponytail không thay TDD — chúng bổ trợ nhau. TDD cho behavior đúng, Ponytail cho scope nhỏ.
