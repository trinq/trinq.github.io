Core Architecture

Supervisor Orchestrator
Single entry point. Routes to sub-agents. Manages both short-term (session) and long-term (persistent) memory.
  • Classifies user intent → routes to correct sub-graph
  • Knows when to invoke human-in-the-loop
  • Aggregates sub-agent responses
always
Sub-Agent Types
Structured Data Agent — NL → SQL/API → summarize tabular result.
structuredcommon

Unstructured Doc Agent (RAG) — vectorized docs → retrieve → synthesize.
unstructuredcommon

Analytics Agent — ReAct + API tools for simple; text-to-code + human review for complex.
analyticscomplex
Intent-Based Sub-Graphs
Separate workflow graphs per intent category (e.g., general QA vs. specific fund questions). Each subgraph is tunable independently.

Changes to the general QA flow don't risk breaking the fund-specific flow.
scale

Quality Nodes

Personalization Node
Same data, different presentation. Tailors depth and framing based on user role (e.g., advisor needs summary, specialist needs detail).
post-retrievalhigh-value
Reflection Node
LLM-as-judge validates answer quality before return. Retry on failure.
gatealways
Summarization Node
Wraps conversation. Updates memory. Returns final answer.
terminalalways

Evaluation Patterns

Start Evaluation Before Ground Truth
Use LLM-as-judge and trajectory evaluation from day one. Ground truth accumulates from review cycles, not the other way around.
"Once you start doing evaluation, you will have review. Once you start doing review, you will accumulate more of the ground truth examples."
— Zheng Xue, JP Morgan
key
Evaluate Sub-Agents Independently
Each sub-agent has its own evaluation dataset and metrics. The structured data agent is evaluated on query accuracy; the doc agent on retrieval precision.
keyindependent

LLM-as-Judge at Scale
Automates quality review without overloading human SMEs. Human review reserved for edge cases. Best when combined with accumulating ground truth from prior reviews.

Build Philosophy

Start Simple, Refactor Often
Day 1: vanilla ReAct agent. Day 2: add RAG agent. Day 3: supervisor + 2 sub-agents. Day 4+: add sub-graphs per intent. Each iteration builds confidence before adding complexity.
key
Human-in-the-Loop is a Design Choice
Not a workaround for unreliable AI. A deliberate architectural decision for high-stakes domains. The system knows when to escalate.

Triggers: ambiguous answers, high financial stakes, out-of-distribution queries, configurable confidence thresholds.
keyhigh-stakes

Data Pre-Processing

Structured Data
Already in databases. Agent translates NL → SQL/API. LLM summarizes result.
low-friction
Unstructured Data
Documents must be pre-processed: ingest → chunk → vectorize → store in vector DB. Only then can RAG agents query at runtime.
upfront cost
Audio / Video
Transcription → treated as unstructured data → same RAG pipeline.
pre-process
