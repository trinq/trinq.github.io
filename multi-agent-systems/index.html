Multi-Agent Systems

This course teaches you to design, build, and improve enterprise multi-agent AI systems. You will learn the Ask David architecture from JP Morgan, how to split work across specialized sub-agents, how to evaluate every sub-agent independently, how to design human-in-the-loop layers, and the empirical accuracy trajectory from 0% to production-grade.

Ask David — The Architecture of an Enterprise Multi-Agent System

"D.A.V.I.D. — Data Analytics Visualization Insights and Decision Making Assistant." When a financial advisor asks "Why was this fund terminated?", a multi-agent system answers — with human oversight, personalized for the audience, and evaluated at every step.

The Supervisor + Sub-Agent Pattern — How to Split Work Across Agents

Lesson 0001 showed you the architecture. This lesson shows you how to design it — where to draw agent boundaries, what each sub-agent actually does, and how the supervisor delegates without becoming a bottleneck.

Lesson 0003 multi-agent-systems 2026-07-04

Evaluation-Driven Development — Measure Before You Build, Improve What You Measure

"Once you start doing evaluation, you will have review. Once you start doing review, you will accumulate more of the ground truth examples." — Zheng Xue, JP Morgan. Design evaluation plans for every sub-agent before you start building.

Lesson 0004 multi-agent-systems 2026-07-04

Human-in-the-Loop Design — When and How to Keep Humans in the Decision

"The last mile is always the hardest mile." HITL is not a workaround for unreliable AI — it is a deliberate architectural decision that says: some decisions carry stakes that justify a human's time.

The 0-to-90-to-100 Accuracy Trajectory — How Real AI Systems Actually Improve

The empirical accuracy trajectory: Phase 1 (0–50%, base model), Phase 2 (50–80%, engineering wins), Phase 3 (80–90%, subgraph isolation), Phase 4 (90–100%, last-mile targeted fixes). The three meta-learnings that govern everything.
