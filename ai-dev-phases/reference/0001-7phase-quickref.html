7-Phase AI Development — Quick Reference

Manager's desk card · ai-dev-phases curriculum

P1 Idea
Owner Manager
Output 1-sentence idea with user + measurable baseline
Gate User named · Problem stated · Baseline measurable
Primary FM Vague ambition — no user, no baseline
"An idea without a measurable baseline is not an idea."
P2 Research
Owner Manager + Engineer
Output research.md with evidence, sources, freshness date
Gate Hard questions from P1 answered · Evidence cited
Primary FM Confirmation search — finds evidence for the answer it wants
"Cache it in a research.md asset."
P3 Prototype
Owner Engineer
Output Throwaway artifact · One question answered
Gate Question answered · Artifact deleted or isolated
Primary FM Production prototype — never gets thrown away
"You can actually commit the prototype."
P4 PRD
Owner Manager
Output PRD doc: outcomes (not solutions) + non-goals + testable requirements
Gate All requirements testable · Non-goals stated · Owner signed
Primary FM Missing non-goals — scope creep has no defense
"A requirement without a test is not a requirement."
P5 Impl Plan
Owner Engineer
Output M-sized steps (2–4 hrs each), each mapped to a PRD requirement
Gate Every step ≤ 4 hrs · Every step traces to a PRD req
Primary FM L-sized steps — too large to supervise, too late to fix
"If you can't write the plan, you can't run the execution."
P6 Execution
Owner Engineer + Agent
Output Merged PRs · Execution log · Deviation reports
Gate All steps complete · Deviations logged · Loopback detector clear
Primary FM Agent runs without supervision tier — deviations hidden
"Run AFK — with the right protocols in place."
P7 QA
Owner QA + Manager
Output QA report: verdict (ship / fix / rollback) + issue log
Gate Binary verdict only · No "ship with known issues"
Primary FM Happy-path only — edge cases ship to production
"Verify — do not trust."
↺ Loopback

When a phase receives evidence that contradicts its assumptions:

Early (2x cost) → Loop to phase it came from

Mid (5x cost) → Loop to P5 — revise impl plan

Late (10x cost) → Loop to P4 — revise PRD

Rule: detect early, loop to the nearest valid phase, update artifact before continuing

Supervision Tiers (P6 Execution)
Tier 1 — Supervised

Agent shows all output before committing. Human reviews every step. Used for unfamiliar patterns, new codebases, risky changes.

Tier 2 — AFK-able

Agent runs unsupervised. Checkpoint at each impl plan step. Deviation report on any approach change. Used for familiar patterns.

Tier 3 — Delegated

Full delegation. Agent runs full feature. Trust elevation requires 5 clean Tier 2 checkpoints on that pattern. Context <60% full.
