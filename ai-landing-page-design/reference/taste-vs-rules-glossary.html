Taste vs Rules — Glossary

Five load-bearing terms used across this course. Keep these definitions stable; if a new term earns load-bearing status, add a row here, never inline-only.

taste

The judgment that distinguishes "technically correct" from "genuinely good." The user holds it; AI does not. Where the page is meaningful, intentional, original — that is taste.

slop

Generic, patterned output that follows rules but lacks intent. Centred hero + 3-column features + gradient CTA + glassmorphism cards can be slop if they don't reflect the brief. If you saw it on every AI-built site last month, suspect slop.

baseline

The first-build reference version, usually produced with Method 1 (Design Skill). The baseline exists to surface slop patterns early, not to ship.

principle

The reason a reference is on your board. Always a verb phrase ("earn the negative space", "use motion to acknowledge, not decorate") rather than a noun ("hero section", "typography").

intent

A short paragraph that names what a component must do for the audience before any prompt is written. If you can't write intent, you don't know what you're building yet.
