What is a cognitive tool?

The frame Judy Fan's whole research program hangs on — and the question it asks about every artifact your agent produces or consumes.
~6 min Lesson 0001 glossary · taxonomy

Judy Fan opens her talk the way a good math teacher would: with the number line.

"Nature didn't give us the number line. We invented it. As Antoni Gaudí once put it, there are no straight lines or sharp corners in nature, but that didn't stop us. We created them anyway."
— Judy Fan, MIT BCS Colloquium, 03:52–05:15

The number line is not a fact about the world. It is a cognitive tool — an external representation that, once you have it, lets you think thoughts you couldn't think before. Extend it into two dimensions and you get the Cartesian coordinate system. Extend that and you get the algebra-geometric bridge that, in Fan's framing, took humans to the moon.

Every cognitive tool in Fan's frame passes the same test: does this representation change what its user can think about?

The test for any cognitive tool: what new questions does it make answerable that weren't before? The number line makes ordering answerable. Cartesian coordinates make intersection-of-curves answerable. A timeline chart makes "how has this changed over time" answerable. A box plot makes distribution-comparison answerable.

Three examples Fan uses to set the frame

From the first ten minutes of the talk (03:52–13:00):

  • Darwin's finches (John Gould's drawings). Side-by-side illustrations are what makes the morphological variation visible. Without the comparison panel, the differences don't pop. The panel is the cognitive tool, not the bird.
  • Galileo's telescope. An optical tool that resolves detail invisible to the naked eye. Cognitive tool by direct analogy: it extends what its user can perceive.
  • Feynman diagrams. Subatomic particles wink in and out of existence — events we will never see with our eyes. The diagram doesn't show us the event. It shows us a calculational handle that lets us reason about it. (Richard Feynman's own words, on the diagrams' invention: "I invented them to help me think.")

All three pass the test. All three make questions answerable that weren't answerable before. None of them are just data — they are structured for a particular kind of question.

Drawing is the original cognitive tool

Fan spends the rest of Part 1 on the most ancient cognitive tool we have: freehand drawing. 30,000–80,000 years ago, anatomically modern humans started making marks on cave walls. The marks are not the thing — they are a tool that lets the maker communicate what they see, and the viewer recover what was seen.

This is where the frame turns useful for agent design. An agent asked to "explain the system" has access to many representations — prose, bullet lists, sketches, diagrams, Mermaid, screenshots. The frame's question is: which one makes the question the user actually has answerable?

Heuristic for agent output format. Don't ask "what does this look like in markdown?" Ask "what question is the user trying to answer, and which representation makes that question answerable for the agent's downstream consumer?" A system-architecture sketch makes "what calls what" answerable. A sequence diagram makes "what happens during the race condition" answerable. A bulleted list makes "what are the components" answerable. The choice is upstream of the syntax.

The "closing the loop" frame Fan closes the talk with

The two halves of Fan's research program — drawing as cognitive tool, and data viz as cognitive tool — share one thesis: humans don't just discover useful abstractions, they apply them to make new things. The discovery-and-engineering loop is what Fan calls the "closing the loop" frame.

"If I had to state it really bluntly, research in my group aims to close this loop — to develop psychological theories that explain how we go about discovering useful abstractions that explain how the world works, jointly with theories that explain how we then apply those abstractions to go make new things."
— Judy Fan, 08:00

Agents have the same loop. Parse a chart → use what you parsed to act on the world. Render a diagram → check that it communicates what the next tool in the pipeline needs to know. The agent doesn't need to "think" — it needs the representation that makes the next step cheap.

What this lesson leaves you with

One frame, three applications:

  1. The cognitive-tool test. "What new question does this representation make answerable?" Use it before you write the agent prompt or pick the output format.
  2. The agent-design reading. Any artifact an agent produces (or consumes) is a candidate cognitive tool. The question is whether it's shaped for the question that follows it.
  3. The discovery-and-engineering loop. Don't stop at "the agent can render this." Ask: "and then what?" The answer to that question is what makes the representation load-bearing.

Try it now

Quick check: pick the cognitive tool

Your agent has finished a long task. The user is about to ask "did this work?" The agent must produce one output that will be used by the user to judge success. Which is shaped most like a cognitive tool for that question?

  2. A pass/fail table is shaped for the question "did this work?" — it makes each criterion answerable at a glance. Prose makes the user re-derive the answer. The table is the cognitive tool.

  2. A Jira board is a coordination artifact, not a cognitive tool. It doesn't make a new question answerable — it makes the same question ("who's doing what") answerable to a different audience (the team). Cognitive tools in Fan's frame change what their user can think about, not just who can see it.

Action items

  1. Audit one agent's outputs. Pick one agent you actually run. Look at its last five outputs. For each, ask: what question does the user have next? Is the current output shaped for that question? If not, the output is a missing cognitive tool, not just a formatting choice.
  2. Pick one artifact to upgrade. Find the output that's most often a "the user has to re-derive it" output (a wall of text where they want a comparison, a screenshot where they want a delta, etc.). Replace it with the representation shaped for the question. Run it for a week.
