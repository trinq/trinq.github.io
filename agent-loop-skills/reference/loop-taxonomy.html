Autoresearch ML

Iterate on an ML artifact (model, config, data pipeline) against a metric. Propose → train → score → keep/revert → log.

karpathy Minimal baseline

Propose one change, train, keep if metric improves, loop. Faithful to Karpathy's original idea. Start here.

Use when: You have a training script + metric and want a minimal, reliable loop. No frills, no analysis.
ml-autoresearch multi-role Diagnostic-first

Reads inside each run: gradient flow, dead neurons, loss curve. Grounds next change in evidence, not guesswork. Optional literature-dial adds paper-grounded changes.

Use when: Training is slow/expensive and you want every change traced to a measured cause.
exploratory-autoresearch Broad exploration

Temperature/swing scheduler forces broad exploration. Escapes hill-climbing one idea forever.

Use when: You're stuck on a plateau and need the loop to try far-out ideas, not just incremental changes.
tournament-autoresearch multi-role Competitive

N agents pitch competing changes; a self-calibrating judge picks one, runs it, and recalibrates by comparing predicted vs realized gain.

Use when: You want the loop to self-correct judge bias over time. Best for escaping single-thread plateaus.
dueling-autoresearch multi-role Parallel racing

Two approaches race the same metric in parallel and borrow ideas across lanes.

Use when: You have two competing architectures/strategies and want them to compete + cross-pollinate.
alpha-evolve multi-role Population-based

MAP-Elites + island model. Diff-mutate + cascade-eval. Inspired by AlphaEvolve paper (Novikov et al., 2025).

Use when: You want evolutionary search over a population, not single-point hill climbing.

Literature & Writing Research

Build, evaluate, and iterate on research artifacts (papers, proposals, figures, writing).

literature-search Shared toolchain

NOT a loop — a shared tool. Paper discovery, snippets, citation graph, full-text over Semantic Scholar + arXiv. Used by other research loops.

Use when: Any research loop needs to find papers. Installed alongside, degrades to WebSearch if absent.
literature-survey Evidence matrix

Builds a saturating evidence/contradiction matrix: sources × claims. Stops when evidence is saturated.

Use when: You need to map what the literature says about a set of claims across many papers.
research-question Question sharpening

Sharpens vague topics into strong, novel, feasible research questions. Web novelty checks pivot already-answered questions toward open sub-problems.

Use when: You have a broad research interest and need to find the specific unanswered question.
hypothesis-gen multi-role Generate + vet

Generates a pool of hypotheses, then literature-vets each one. Keeps only those that survive the literature check.

Use when: You need many hypotheses generated and filtered against existing work.
research-proposal multi-role Grade + revise

ScholarEval grades a proposal against the literature. Judge + Reviser iterate until the score clears a bar (e.g. 45 → 84).

Use when: You have a draft proposal and want it graded + revised to pass peer review bar.
scientific-writer multi-role Peer review loop

Specialist judges + independent peer-reviewer critique a draft; a writer revises until the score clears a bar.

Use when: You want adversarial peer review of a scientific manuscript before submission.
scientific-figure multi-role Figure iteration

Generator drafts a figure from data/brief; adversarial critic grades it against a fixed rubric. Both can consult literature. Iterates until paper-ready (80 → 96).

Use when: You need a publication-quality figure that survives scrutiny from reviewers.

Data Analysis

Work with datasets: find findings, verify claims, clean tables, diagnose anomalies.

data-analysis Hypothesis → verify

Hypothesis → verify discovery. Every finding backed by a reproduced number at a meaningful effect size. Correctly refuted 2 planted wrong findings in testing.

Use when: You have a dataset and want to surface real findings, not hallucinations. Each finding is reproduced + effect-sized.
anomaly-investigation Diagnose anomalies

Diagnoses the cause of a known anomaly by forming, testing, and eliminating candidates. Think: "why did X suddenly spike last Tuesday?"

Use when: Something went wrong and you need to find the root cause, not just the symptom.
claim-verify Adversarial verification

Extracts checkable claims from a draft, reproduces each number, stress-tests against threats (outliers, confounds, Simpson's reversals). Marks verified/fragile/refuted.

Use when: You have a results draft ready to publish and want to red-team every claim before it goes out.
tabular-cleanup Schema inference

Cleans a messy table to an inferred data contract with deterministic checks. Loop-until-clean: all checks pass.

Use when: You have a dirty CSV/table and want to clean it with a reproducible schema, not manual editing.

Code & Optimization Dev

Iterate on code, queries, and prompts with correctness gates and optimization targets.

optimize-loop Correctness-gated optimization

Pluggable correctness gate + minimized metric. Two modes: code (tests green + complexity ↓) and SQL (identical results + latency ↓). Real run: 1,131ms → 1.055ms (~1,073×).

Use when: You want to make code faster or cleaner without breaking correctness. This is the most generally useful loop for developers.
prompt-optimize Black-box oracle

Evolves a prompt against a user-supplied scoring command (black-box oracle). No internal access to the scoring — just input/output.

Use when: You have a prompt and an evaluator (human, test suite, another model) and want to evolve the prompt toward better outputs.

Other Specialized

red-team Adversarial, loop-until-dry

Surfaces distinct failure classes of a system you own. Loop-until-dry: no new failure classes found for K rounds. Surfaces root-cause + one-line fix for each.

Use when: You own a system (content filter, classifier, auth) and want to find its weaknesses before attackers do. (Authorized testing only.)
power-analysis Experiment sizing

Sizes and pre-registers a two-arm experiment to hit a target statistical power. Solves n = 100/group for 80% power via Monte-Carlo, fixes validity flaws.

Use when: You're designing an A/B test or experiment and need to know how many samples to collect before you start.
