Lesson 0003
Grill Me: Reach Shared Understanding Before You Build

Before you write a single line of code or a single skill — walk down the entire design tree together with the AI. This lesson teaches the most impactful 3-sentence skill in the entire workflow.

The Problem It Solves

You've probably experienced this: you go into Plan mode, describe your idea briefly, and the AI immediately spits out a full implementation plan. It looks thorough. But then you implement it and things are wrong in ways that feel obvious in hindsight — wrong page, wrong API shape, wrong scope. You trusted the AI to design something it didn't understand.

The root cause: you never reached shared understanding. The AI was working from a vague approximation of your intent, not from a deep model of what you actually wanted.

The grill-me skill solves this. It's three sentences. It can take a 30–50 question conversation. And it's the highest-leverage skill in the entire workflow.

The Skill

grill-me — 3 sentences

Interview the user relentlessly about every aspect of this plan until we reach a shared understanding. Walk down each branch of the design tree, resolving dependencies between decisions one by one. If a question can be answered by exploring the codebase, explore the codebase instead.

That's it. Three sentences. No examples, no step-by-step instructions. The power is in the specificity of each sentence.

The Design Tree

The concept comes from Frederick P. Brooks' The Design of Design. The idea: every design decision is a branch on a tree. Below each branch are more branches — sub-decisions that depend on the parent decision. To understand a design fully, you must walk down every path.

Example: say you want a "search page."

  • Search page
    • Text box only
      • Search syntax: simple keywords or advanced query language?
      • Results: paginated or infinite scroll?
    • Advanced search
      • Filters: what fields? Date range? Tags? Author?
      • Sort options: relevance, date, alphabetical?
      • Save searches for logged-in users?

If you choose "text box only," you don't need to design filters. If you choose "advanced search," every filter you add creates more sub-questions. The tree grows until the design is as complete as it can be before you commit to code.

Most engineers do this intuitively. The problem with AI agents: they don't. They pick a branch prematurely and start building. The grill-me skill forces the conversation.

What "Relentless" Actually Looks Like

Here's a real grilling session from the author's course. The AI asked 16 questions before a single line of code was written. For a feature that ended up being a 4-day implementation.

The conversation started with: "Here's some research I did in this markdown file. Grill me — I'd like to add this to the right page."

Output: 16 questions. The AI explored the codebase first, then systematically walked through: where does the document live → what's the UI layout → which modes get the document panel → document lifecycle → what does the right document tool look like → edit tool shape → ... all the way to question 16.

Some sessions run 30–45 minutes. That's normal. The time spent questioning is always less than the time wasted rebuilding something that was designed wrong.

Rule 3: Explore the Codebase Instead

The third sentence is subtle but important:

If a question can be answered by exploring the codebase, explore the codebase instead.

This prevents useless back-and-forth. Don't ask the user "where is the auth module?" — just open the codebase and find it. The AI should use its tools, not the user's memory.

Why It Matters for Loop Skills

When you write an Agent Loop Skill, you're encoding a process. If you encode a process you don't fully understand, the loop will do the wrong thing confidently. You'll spend more time debugging the loop than you would have spent writing it manually.

The grill-me skill forces you to think through:

  • What is the termination condition? When does the loop know it's done?
  • What is the signal contract? What does each pass produce and consume?
  • What are the failure modes? What goes wrong, and how does the loop recover?
  • Who is the oracle? Who or what decides if an output is good?

These are the questions that make or break a loop. Use grill-me before every new skill.

When to Use It

  • Before writing any SKILL.md — especially for a new loop type
  • Before a major refactor — make sure you agree on the destination
  • When the AI starts building too fast — invoke it to hit the brakes
  • When you have a vague idea — "I want to automate X" is not a spec

Warning: grill-me can feel uncomfortable. It means sitting with uncertainty longer than you want to. Resist the urge to jump to solutions. The discomfort pays off in better-designed loops.

Action Items

  • Try it now Take your create-blog-post SKILL.md and say "grill me" to me. Let's see how many questions it takes before we reach shared understanding.
  • Record what you learned After the session, note which questions revealed hidden assumptions. That becomes your design tree.
