Lesson 0002

Install & Pick Your Loop

Install loops into Cursor, then pick a real task to map onto a loop

Step 1 — Clone the repo

First, clone the Agent Loop Skills repo to your machine. One time only.

1Clone repo
Run this in your terminal:
git clone https://github.com/gaasher/agent-loop-skills.git ~/code/agent-loop-skills

Step 2 — Copy into Cursor's skills folder

Cursor uses the same format as Claude Code and Codex — just copy the loops/ folder.

2Copy all loops into Cursor skills
Command:
cp -r ~/code/agent-loop-skills/loops/* ~/.agents/skills/

Check if ~/.agents/skills/ exists first. If not, create it first:

mkdir -p ~/.agents/skills && cp -r ~/code/agent-loop-skills/loops/* ~/.agents/skills/
3Verify installation
List the folder to confirm:
ls ~/.agents/skills/ | head -20
You should see: karpathy/, optimize-loop/, claim-verify/, etc. Each folder contains a SKILL.md inside.

Your Task: Map It to a Loop

From MISSION.md, your goal is to automate repetitive workflows. Look at the tasks you do daily and map them:

  • Writing and maintaining a blog (trinq.github.io) Convert HTML lesson → redirect post every time you publish. Each post: extract title, description, create YAML frontmatter, commit & push. 📝 claim-verify? data-analysis? Write a new loop!
  • Prompt optimization Write prompt → run test → see result → fix prompt → repeat. Want automation: script to measure "prompt score" and find better prompt automatically. → prompt-optimize
  • Clean data / messy tables Dirty CSV, need to normalize schema, fill missing values, fix types. One run and it's done. → tabular-cleanup
  • Research: find papers + synthesize claims Search web → read abstract → extract claims → compare conflicts. Repeat until saturated. → literature-survey
  • Debug/refactor code Measure test coverage + cyclomatic complexity. Improve both → keep, don't → revert. → optimize-loop (mode: code)
  • Verify claims from a dataset There's an article with numbers. Want to check: are those numbers correct, are there confounders. → claim-verify

Your turn. In your workspace, which task is the most realistic to start with? Pick 1. The first thing we'll do is set up its bindings — write a loop.run.yaml — then run that loop.

Quick Map: Task → Loop

If you already know which task you want to automate, use this table:

🐢
karpathy
Minimal. Train model, keep if metric better. Runs immediately, minimal config.
⚡
optimize-loop (code mode)
Refactor code. Correctness gate (tests green) + complexity/metrics down. Best for dev work.
🔤
prompt-optimize
Evolve prompt against a scoring command. User-provided oracle. Need to write a scorer first.
📊
data-analysis
Hypothesis → verify with effect size. No hallucinations (tested). Good for existing data.
✅
claim-verify
An article with claims, verify each number. Stress-test with outliers/confounds. Publish-safe.
🗂️
tabular-cleanup
Clean messy table. Infer schema + deterministic checks. One run and done.
🔬
literature-search / literature-survey
Research tool. Find papers, build evidence matrix. Needs API keys setup (Semantic Scholar, arXiv).
🛡️
red-team
Adversarial testing. Find failure modes. Use when you own the system being tested.

Your Blog Task

The blog task (HTML → redirect post) seems like the most interesting candidate — because your current create-blog-post skill is a single run. If you want to improve it over time (detect if already posted, auto-select category, write better descriptions...), that's a candidate for a loop.

However, the blog task may not fit an existing loop. You might need to write a new loop — Lesson 3 will teach you how to write a loop from scratch based on a spec.

Before writing a new loop, Lesson 3 will practice with an existing loop first — so you can see how a loop actually runs in Cursor.
