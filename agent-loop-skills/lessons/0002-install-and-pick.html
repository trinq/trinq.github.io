Lesson 0002

Install & Pick Your Loop

Cài loops vào Cursor, chọn task thực tế để map vào loop

Bước 1 — Clone repo

Đầu tiên, clone Agent Loop Skills về máy. Một lần thôi.

1Clone repo
Chạy lệnh này trong terminal:
git clone https://github.com/gaasher/agent-loop-skills.git ~/code/agent-loop-skills

Bước 2 — Copy vào skills folder của Cursor

Cursor dùng cùng định dạng với Claude Code và Codex — chỉ cần copy folder loops/.

2Copy tất cả loops vào Cursor skills
Lệnh:
cp -r ~/code/agent-loop-skills/loops/* ~/.agents/skills/

Kiểm tra xem folder ~/.agents/skills/ có tồn tại chưa. Nếu chưa, tạo trước:

mkdir -p ~/.agents/skills && cp -r ~/code/agent-loop-skills/loops/* ~/.agents/skills/
3Verify đã cài đúng
List folder để xem:
ls ~/.agents/skills/ | head -20
Bạn sẽ thấy: karpathy/, optimize-loop/, claim-verify/, v.v. Mỗi folder có SKILL.md bên trong.

Task của bạn: map vào loop

Từ MISSION.md, mục tiêu của bạn là tự động hóa workflow lặp đi lặp lại. Hãy nhìn vào các task bạn làm hàng ngày và map chúng:

  • Viết và duy trì blog (trinq.github.io) Chuyển HTML lesson → redirect post mỗi khi publish bài mới. Mỗi bài: trích title, description, tạo file YAML frontmatter, commit & push. 📝 claim-verify? data-analysis? Viết loop mới!
  • Tối ưu prompt Viết prompt → chạy test → xem kết quả → sửa prompt → lặp. Muốn automated: script đo "prompt score" và tự tìm prompt tốt hơn. → prompt-optimize
  • Clean data / table messy CSV bẩn, cần chuẩn hóa schema, điền missing values, fix types. Chạy 1 lần là xong. → tabular-cleanup
  • Research: tìm papers + tổng hợp claims Search web → đọc abstract → trích claims → so sánh xung đột. Lặp đến khi saturated. → literature-survey
  • Debug/refactor code Đo test coverage + cyclomatic complexity. Cải thiện cả hai → giữ, không → revert. → optimize-loop (mode: code)
  • Verify claims từ dataset Có bài viết với các số liệu. Muốn check: số đó có đúng không, có bị confounders không. → claim-verify

Your turn. Trong workspace của bạn, task nào là thực tế nhất để bắt đầu? Chọn 1. Việc đầu tiên chúng ta sẽ làm là đặt binding cho nó — viết loop.run.yaml — rồi chạy loop đó.

Map nhanh: task → loop

Nếu bạn biết task mình muốn tự động hóa, dùng bảng này:

🐢
karpathy
Minimal. Train model, keep if metric better. Chạy được ngay, ít config.
⚡
optimize-loop (code mode)
Refactor code. Correctness gate (tests green) + complexity/metrics down. Tốt nhất cho dev work.
🔤
prompt-optimize
Evolve prompt against a scoring command. User-provided oracle. Cần viết scorer trước.
📊
data-analysis
Hypothesis → verify với effect size. Không hallucinations (tested). Tốt cho data có sẵn.
✅
claim-verify
Một bài viết có claims, verify từng số. Stress-test với outliers/confounds. Publish-safe.
🗂️
tabular-cleanup
Clean messy table. Infer schema + deterministic checks. Chạy 1 lần là done.
🔬
literature-search / literature-survey
Research tool. Tìm papers, build evidence matrix. Cần setup API keys (Semantic Scholar, arXiv).
🛡️
red-team
Adversarial testing. Tìm failure modes. Dùng khi bạn sở hữu hệ thống cần test.

Task blog của bạn

Task blog (HTML → redirect post) có vẻ là ứng cử viên thú vị nhất — vì create-blog-post skill hiện tại của bạn là một lần chạy. Nếu muốn cải thiện nó theo thời gian (detect đã post chưa, tự chọn category, tự viết description tốt hơn...), đó là candidate cho một loop.

Tuy nhiên, task blog chưa chắc đã fit loop có sẵn. Có thể cần viết loop mới — Lesson 3 sẽ học cách viết loop từ đầu theo spec.

Trước khi viết loop mới, Lesson 3 sẽ thực hành với một loop có sẵn trước — để bạn thấy cách một loop chạy thực tế trong Cursor.
