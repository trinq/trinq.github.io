Interactions and Motion

Lesson 0008 · ~7 minutes · Skill

Static design is half a design. The other half is what happens when users interact with it. This lesson covers the principles and timing that make interfaces feel alive — without feeling chaotic.

The Hidden Layer
Motion is not decoration. It communicates: this is clickable, this is loading, this just happened. Use it to clarify, not to impress.

What Motion Communicates

Every transition carries meaning. Before adding animation, ask what it should communicate:

  • Affordance — "I'm clickable" (hover states, button presses)
  • Feedback — "Your action registered" (form submits, toggle states)
  • Hierarchy — "This is the new state" (modal openings, page transitions)
  • Continuity — "You're in the same place" (subtle scrolling, parallax)
  • Delight — "Someone cared about this" (considered micro-interactions)

If you can't name the communication, you probably don't need the motion.

Duration: How Long

Duration determines whether motion feels responsive or sluggish. Use these ranges:

Duration Best for
0-100ms Instant feedback (button press, hover color change)
150-250ms Subtle state changes (dropdown open, tooltip)
250-400ms Component transitions (modal, drawer, accordion)
400-600ms Page-level transitions (route change, content swap)
600ms+ Decorative or choreographed (rare, used sparingly)

The shorter the distance traveled, the shorter the duration. The further or more dramatic the change, the longer it takes — but never longer than necessary.

Easing: How It Moves

Linear motion feels mechanical. Easing makes motion feel physical and natural. These four cover most needs:

Ease-out
Arrives quickly, settles gently. Most common.
cubic-bezier(0.0, 0.0, 0.2, 1)
Ease-in
Starts slow, accelerates. For exits.
cubic-bezier(0.4, 0.0, 1, 1)
Ease-in-out
Smooth both directions. For symmetric transitions.
cubic-bezier(0.4, 0.0, 0.2, 1)
Spring/overshoot
Playful, expressive. Use sparingly.
Material: spring(stiffness, damping)

Rule of thumb: things entering the screen use ease-out (they arrive). Things leaving use ease-in (they accelerate away). This mirrors physical reality and feels natural.

The Hover State Spectrum

Hover is the most common interaction. Its intensity should match the action's importance:

  • Links — Underline appears, color shift, 0ms (instant)
  • Buttons (primary) — Subtle background darken, 100-150ms
  • Buttons (secondary) — Border or background fill, 150ms
  • Cards — Slight lift (translateY(-2px)) + shadow growth, 200ms
  • Icon buttons — Background fade-in, 150ms

Higher-stakes actions (primary buttons) deserve more pronounced feedback. Secondary actions stay subtle.

Transitions to Master

Every product uses the same core transitions. Get these right:

Modal opening — Background fades, modal scales from 95% to 100%, 250ms, ease-out. Optional backdrop blur.

Drawer / sidebar — Slides in from edge, 300ms, ease-out. Content (not background) is what moves.

Accordion expand — Height auto animates, 250ms, ease-in-out. Avoid display: none jumps.

Toast / notification — Slides in from corner, 300ms. Auto-dismisses after 4-6 seconds with same exit.

Page transition — Outgoing fades to 0, incoming fades from 0, 300ms total. Keep it brief; users want to move on.

Skeleton loading — Subtle shimmer animation, infinite, 1.5s cycle. Indicates content is coming.

Scroll-Triggered Motion

As users scroll, content can respond. Use this sparingly:

  • Fade-in on enter — Element appears as it scrolls into view. Most useful for long pages.
  • Stagger children — Lists/cards fade in one after another (50ms stagger). Adds rhythm without noise.
  • Parallax — Background moves slower than foreground. Use very subtly; overuse causes motion sickness.

Avoid: animations that block scrolling, animations that replay when scrolling back, anything that makes users wait.

The Scrolling Test
If motion makes scrolling feel slower or harder, you've overdone it. Motion should support navigation, not fight it.

Accessibility Considerations

Motion can disable or harm users with vestibular disorders. Always respect:

  • prefers-reduced-motion — Disable non-essential animations for these users
  • Never rely on motion alone to communicate (also use color, text, or position)
  • Keep loops short or provide a way to stop them
  • Avoid large-scale parallax or rapid movement

This isn't optional — it's part of good design.

The AI Prompting Angle

When prompting AI for components, motion descriptions matter. Include:

  • "Subtle hover lift, 200ms ease-out"
  • "Modal scales from 95% to 100% on open"
  • "Transitions should feel calm, never bouncy"
  • "Respect prefers-reduced-motion"

Without these, AI generates either no motion or generic motion. Both miss your intent.

Common Motion Mistakes

  • Too long — Anything over 500ms feels sluggish for UI. Save longer durations for video.
  • Too bouncy — Spring/elastic easing feels playful but quickly feels amateur in serious products.
  • Too much — Animating every element creates noise. Animate only what matters.
  • Wrong easing — Linear motion in a sophisticated product feels broken.
  • No reduced-motion support — Accessibility violation that excludes real users.

Exercise: Audit and Improve Motion

Practice · 30 minutes
Audit motion in your reference library.

Step 1: Pick 3 product references from your library (Lesson 0005).

Step 2: For each, note what motion is used: hover, modal, page transition, scroll.

Step 3: Estimate durations and easings (visually).

Step 4: What feeling does the motion create? Calm? Responsive? Playful?

Step 5: Apply to your own product: which transitions matter most? What duration? What easing?

Save your motion spec — it's a core part of your design system.

Key Takeaways

  • Motion communicates; always know what it's saying
  • Match duration to distance and importance
  • Use ease-out for arrivals, ease-in for exits
  • Hover intensity should match action importance
  • Respect prefers-reduced-motion for accessibility
  • Less motion, done well, beats more motion
