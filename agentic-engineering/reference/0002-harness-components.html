Diagnostic quick-reference

Agent forgets what it did 3 steps ago
→ Component 4 (Context Management)
Tool calls fail silently
→ Component 8 (Error Handling)
Context fills up with garbage
→ Component 4 (Context Management)
Agent ignores your instructions
→ Component 5 (Prompt Construction — check position in context window)
Agent does too much / goes out of scope
→ Component 9 (Guardrails) or Component 2 (Tool Scoping)
Code is correct but semantically wrong
→ Component 10 (Verification Loops — add LLM-as-judge)
Agent gets stuck in retry loops
→ Component 8 (Error Handling — cap retries)
Quality degrades in long sessions
→ Component 4 (Context Management — trigger compaction earlier)

Seven architectural decisions

#DecisionDefault for solo Claude Code
1Single vs. multi-agentSingle (split at ~10 overlapping tools)
2ReAct vs. plan-and-executePlan-and-execute at workflow, ReAct at step
3Context window strategyCompaction + JIT retrieval
4Verification loop designTests (deterministic) + review agent (semantic)
5Permission architecturePermissive inside sandbox
6Tool scopingLazy loading, minimum needed
7Harness thicknessThin (bet on model improvement)
