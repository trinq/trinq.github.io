Pitch: Finding Your Unknowns

A working pitch · artifact for the Lesson 5 worked example · self-titled "the launch video equivalent"

I shipped a 5-lesson series on the user-side of agentic coding — the techniques that determine what the model is actually asked. Each lesson is short (~10–14 min), self-contained, and ends in quizzes that test your judgment about the next prompt you'll send, not trivia about the lesson.

The problem it solves

With Fable 5, the model's raw capability is no longer the bottleneck. The bottleneck is the user's ability to surface unknowns before, during, and after implementation. Every unknown left on the table becomes a guess the model has to make on the user's behalf, and every guess can be wrong.

The series is built around Cat Wood's taxonomy (X thread, 4 Jul 2026): a 2×2 of unknowns (known/unknown × known/unknown), with one technique class for each cell. We ship all four classes in five lessons because the fourth class (synthesis) is rich enough for its own lesson.

What makes it different from a typical "how to use Fable" guide

  • It teaches trigger phrases, not metaphors. "Blindspot pass" and "interview me one question at a time" are literal prompt phrases. Paraphrasing produces thinner output; L2 calls this out as the first thing to internalize.
  • It runs live rehearsals in the lesson body. Each technique is demonstrated against a real, unresolved problem — the curriculum arc itself, the lesson plan, the pitch format decision. The demonstrations aren't fabricated.
  • It teaches the artifact format, not just the prompt. L4 teaches the implementation plan as a deliverable (HTML, tweak-points first, mechanical last, named verification). L5 teaches the implementation notes as a deliverable (.md file in the project root, format-stable across runs).
  • It flags inferences as inferences. The "blindspot pass can backfire with strong priors" warning in L2 is not in Wood's text; it surfaced during a self-blindspot and is labeled as such so the reader knows it's a pressure-test item, not a cited claim.

What I'd push back on if you challenge it

  • "Five lessons is a lot." The arc could be four if we collapsed L5 (combine during+post is what we did, but combining pre + during would be too much). Five is the smallest number that preserves Wood's window structure (before/during/after) without padding.
  • "The quizzes test the lesson, not the next prompt." The quizzes are calibrated to the next prompt the user will write — the output of L1 is a reframed prompt; L2's quizzes ask what the right prompt shape is, not what L2 said about prompt shape.
  • "The technical risk is the live rehearsals." If Wood's taxonomy turns out to be incomplete or wrong, the rehearsals age badly because they're recorded in the lesson HTML. That's a real risk. Mitigation: the rehearsal in L3 (curriculum arc) is the one most likely to age; if a future direction emerges, L3's rehearsal shows its seams, not its failure.
