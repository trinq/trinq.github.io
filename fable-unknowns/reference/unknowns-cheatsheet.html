The seven techniques — by time window

Pre-implementation

Before any code is written.
  1. Blindspot pass "Run a blindspot pass on this." → L2
  2. Interview "Interview me one question at a time." → L2
  3. Brainstorm "Make me an HTML page with 4 wildly different design directions." → L3
  4. Prototype "Before wiring anything up, make a single HTML file with fake data." → L3
  5. Reference "Look at <ref> and reimplement X. Tell me what to look for." → L3
  6. Implementation plan "Write a plan in HTML. Lead with tweak-points. Bury mechanical." → L4

During implementation

While the code runs.
  1. Implementation notes "Log this to implementation-notes.md." → L5

Two sections in the notes file: Surprises and decisions — what happened, why. Deviations — what changed vs. the plan.

Post-implementation

After the code ships.
  1. Pitch "Write a pitch: lede, problem, differentiator, push-back." → L5
  2. Explainer "Write the longer explainer for context." → L5
  3. Post-quiz "Quiz me. I only merge after I pass perfectly." → L5

Two sentences to memorize

Pre (L4): Lead with the decisions I'll tweak — data model, type interfaces, user-facing. Bury the mechanical. The plan is a deliverable that travels.
Post (L5): Every surprise during execution deserves a log line; every shipped change deserves an artifact the next person can pick up.
If you only do one thing
Iterate, don't plan. One more loop (blindspot, brainstorm, prototype) is cheaper than getting the prompt wrong the first time. Default to running one more technique, not to writing one more paragraph.
