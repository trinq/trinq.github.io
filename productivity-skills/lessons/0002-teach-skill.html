The teach Skill

Learn anything clearly — the Feynman Technique

To invoke:

/teach [topic]

The Core Idea If you can't explain it simply, you don't understand it well enough. The Feynman Technique: assess first, explain second. Don't teach what you think they know — teach what they actually need.

The Assessment Loop

Assess

What do they know?

→

Bridge

Fill the gaps

→

Explain

Teach the concept

→

Check

Did they get it?

The Dialogue Pattern

/teach me React hooks
Teach mode — assessing first
Let's start with what you already know. When you think of React hooks, what comes to mind?
I know class components and useState a bit, but useEffect confuses me.
Perfect. So you're at useState fluently, useEffect is the gap. Let's build from what you know. useState manages one snapshot of state. useEffect manages... what?
Side effects? Is that like componentDidMount?
Exactly! You've got it. componentDidMount, componentDidUpdate, componentWillUnmount — useEffect is React's way of unifying all three lifecycle methods into one API. But with a twist...

Why Assess First?

✓ Assess First

  • Fill actual gaps
  • Build on existing knowledge
  • Don't waste time on basics
  • Tailor to their level
  • Faster learning

✗ Jump In

  • Teach what they already know
  • Miss critical gaps
  • Boring for experts
  • Confusing for beginners
  • Inefficient

The Gap Analysis

Before explaining any concept, ask:

  • What do they already know? — Build on this
  • What are they trying to do? — Ground in their goal
  • Where are they stuck? — Find the specific gap
If someone says "I know the basics", ask for a specific example. "Tell me what useState does" is better than assuming.

Key Principles

The Prior Knowledge Rule New knowledge sticks to existing knowledge. Without hooks in React, useEffect is abstract. With hooks, useEffect connects to something real.
The Specificity Rule "I don't get it" is useless. "I don't get when to use useEffect vs useMemo" is actionable. Push for specificity.

Example: Teaching Git Rebase

/teach git rebase
Before I explain rebase, tell me: what do you know about git merge?
It combines two branches by making a new commit with both changes.
Perfect. Now imagine you could do that differently — instead of creating a merge commit, what if you could... transplant one branch onto another, as if you started from there?

When to Use /teach

  • Learning a new framework or language
  • Reviewing concepts before an interview
  • Gap analysis — what do you actually know?
  • Teaching others — use the same technique
