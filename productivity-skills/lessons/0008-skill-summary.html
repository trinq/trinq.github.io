Skills Reference

🎓/teach

The meta-skill for learning anything clearly by checking understanding first. Uses the Feynman Technique: if you can't explain it simply, you don't understand it well enough.

When to Use

Learning something new, reviewing fundamentals, gap analysis

Key Principle

Assess before explaining — ask what they know first

The Loop

Assess → Bridge → Explain → Check

🔥/grill-me

The stress-testing skill. Interrogate a plan relentlessly until reaching shared understanding, exposing flaws and assumptions.

When to Use

Before building anything, validating decisions, finding gaps

Key Principle

No idea survives contact with relentless questioning

Questions Cover

Why, what, who, how, edge cases, failure modes

💡/brainstorming

Generate ideas collaboratively with quantity over quality. Defer judgment, build on ideas, welcome wild concepts.

When to Use

Need options, exploring possibilities, creative blocks

Key Principle

"Yes, and..." not "Yes, but..."

Rules

Quantity, build on ideas, wild welcome, no criticism

🛠️/build

Implementation with structure. Plans first, then builds incrementally with verification at each step.

When to Use

Ready to implement, need structure, want checkpoints

Key Principle

Plan, then build — not the other way around

Approach

Incremental builds with verification

🔧/systematic-debugging

Diagnosis loop for hard bugs and performance regressions. Gather evidence, form hypothesis, test systematically.

When to Use

Something broken/throwing/failing/slow

Key Principle

Evidence before hypotheses — never guess

Loop

Gather → Hypothesis → Test → Repeat

🔗/tdd

Test-driven development. Write the test first, watch it fail, then write code to make it pass.

When to Use

Building features, fixing bugs, ensuring quality

Key Principle

Red → Green → Refactor

Benefits

Better design, fewer bugs, living documentation

Core Principles

The Meta-Skill Know where you are. Before choosing a skill, identify your phase: learning, ideation, evaluation, execution, or verification. Match the skill to the phase.
The Combination Rule Chain skills intentionally. The power is in composition. Brainstorm → Grill → Build handles most situations. Add more steps only when needed.
The Judgment Rule Defer evaluation during ideation. Generate first, judge later. "Yes, and..." beats "Yes, but..." during brainstorming.
The Evidence Rule Evidence before hypotheses. Never guess at root causes. Gather data, form hypotheses, test systematically.
