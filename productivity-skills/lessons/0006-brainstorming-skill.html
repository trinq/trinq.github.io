The brainstorming Skill

Generate ideas collaboratively — quantity over quality, no judgment

To invoke:

/brainstorm [topic]

or

/brainstorming [topic]

The Core Idea

Brainstorming is a creative technique for generating many ideas quickly, without judgment. The goal is quantity — the more ideas, the more likely you find a gem. Judgment kills creativity; permission enables it.

The Golden Rule Defer judgment. In brainstorming mode, every idea is valid. Criticism, even "that's interesting but...", is banned. Save evaluation for after.

The Brainstorming Rules

⚡

Go for Quantity

10 ideas is better than 3. The goal is options, not perfection.

🔀

Build on Ideas

"Yes, and..." not "Yes, but..." Stack ideas like building blocks.

🌊

Wild Ideas Welcome

The crazy idea leads to the clever idea. Let go of feasibility temporarily.

🚫

No Criticism

No "that's impossible", "we tried that", or even subtle doubt. Judgment is off.

The Dialogue Pattern

/brainstorm ways to reduce meeting fatigue
Brainstorming mode activated — defer judgment, build on ideas
Let's brainstorm ways to reduce meeting fatigue. What's the core problem?
We have too many recurring meetings and people are burned out.
Got it. Let's generate ideas — no judgment yet. Some directions to explore: - Meeting-free days (Slack Friday) - Async standups instead of synchronous - "Walking meetings" for 1:1s - No-meeting blocks on calendars - AI-generated summaries instead of attending - Mandatory camera-off option - "20% time" protected blocks What resonates? What should we build on?
I like the AI summaries idea. What if we could have an AI attend, take notes, and ask questions on our behalf?
Yes! The AI proxy concept. Building on that: - AI attends, summarizes, flags action items - AI asks pre-submitted questions on your behalf - AI raises hand when your input is critical - AI schedules "micro-check-ins" instead of hour-long meetings What else could we add to the AI proxy angle?
We're building a rich idea set. Ready to evaluate, or generate more first?
Let's evaluate. What are the top 3 most impactful?
Mode shift: brainstorm → evaluate

Brainstorming vs Debating

Brainstorming

  • Generate ideas first
  • "Yes, and..."
  • Defer judgment
  • Build on others
  • Quantity mindset
  • Creative, open

Debating

  • Evaluate as you go
  • "Yes, but..."
  • Criticize constantly
  • Refute others
  • Quality mindset
  • Critical, closed
Criticizing mid-brainstorm

"The AI proxy idea is interesting, but we don't have the budget for that right now."

Budget is valid — but later. Keep generating first.

Example: Building a Skill

Brainstorming a Code Review Skill

  • Idea: Automated linting on PR open
  • Build on: Add auto-formatting too
  • Idea: AI comments with style suggestions
  • Build on: + severity levels (nit, warn, block)
  • Idea: Summary comment at PR close
  • Build on: Diff view of changes over time
  • Wild idea: PR review game with points
  • Build on: Leaderboard, badges, streak counts
  • Idea: Reviewer recommendation based on code ownership

Pro Tips

Start with "How might we..." questions. "How might we reduce meeting fatigue?" is more generative than "How do we reduce meetings?"
Use prompts to push past obvious ideas. "What are 10 completely different approaches?" forces lateral thinking.
Set a timer. 10-15 minutes of pure generation, then evaluate. The clock creates urgency and prevents perfectionism.
Capture everything. Even bad ideas have seeds. Write them all down — you can filter later.
