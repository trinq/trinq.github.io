Writing Great Skills — Foundations

The anatomy of effective agent skills

Before you invoke skills, learn to write them.

Understanding how skills are built makes you better at using them.

The Core Idea A skill is a structured prompt that activates a specific mode of thinking. The best skills are clear, focused, and produce consistent results. Writing great skills is a craft.

What Makes a Skill?

A skill has three essential components:

Trigger

What activates the skill

Instructions

What the skill does

Output

What the skill produces

The Three Components

1. Trigger Required

The invocation pattern that activates the skill. This is how users call the skill into action.

Examples

/teach React hooks
/grill-me the authentication flow
/brainstorm onboarding improvements

2. Instructions Required

Clear, specific guidance on what to do. The instructions define the skill's behavior and constraints.

Good Instruction

Assess the user's current knowledge first. Ask what they 
already know before explaining. Bridge gaps with analogies.
End by checking understanding.

3. Output Format Optional

How the results should be structured. Not always needed, but helpful for consistency.

Example

Output a JSON object with:
- topic: what was taught
- gaps_identified: what needs more work  
- key_takeaways: 3-5 bullet points

Skill File Structure

Typical Skill File

# File: skills/teach/SKILL.md

name: teach
description: Learn by assessing first using the Feynman Technique
triggers:
  - "/teach"
  - "/learn"
  
instructions: |
  Before explaining any concept, assess what the user already knows.
  Ask targeted questions to identify gaps. Build explanations that
  bridge from known to unknown. End by checking understanding.
  
output: |
  Structure output as:
  - Assessed knowledge level
  - Key gaps identified
  - Concept explanation (bridged)
  - Comprehension check

examples:
  - input: "/teach closures"
    behavior: "Ask what they know first"

Writing Principles

✓ Do

  • Be specific about the behavior
  • Give concrete examples
  • State the phase/mode clearly
  • Include constraints when needed
  • Match tone to purpose
  • Test with real usage

✗ Don't

  • Be vague or generic
  • Include every possible case
  • Use conflicting instructions
  • Assume too much context
  • Over-complicate the structure
  • Skip testing

Phases and Modes

Skills work by putting the agent in a specific mode. Each mode has a distinct purpose:

  • Learning mode — absorbing, understanding, gap analysis
  • Ideation mode — generating, exploring, quantity over quality
  • Evaluation mode — questioning, challenging, stress-testing
  • Execution mode — building, implementing, structured delivery
  • Diagnosis mode — investigating, troubleshooting, evidence-based
The Mode Rule One skill, one mode. Don't mix learning and evaluation in the same skill. Keep modes pure for predictable results.

Trigger Design

Triggers should be:

  • Short — one or two words maximum
  • Intuitive — matches what users would naturally type
  • Distinct — doesn't conflict with other skills
  • Flexible — works with or without arguments
The trigger is the contract with users. Make it memorable and intuitive. "/teach" beats "/run-feynman-technique-with-gap-analysis".

Testing Your Skills

Before finalizing a skill, test it:

Skill Testing Checklist

  • Activation: Does the trigger reliably activate the skill?
  • Consistency: Does it behave the same way each time?
  • Clarity: Is the output clear and actionable?
  • Edge cases: Does it handle unexpected inputs gracefully?
  • Mode purity: Does it stay in one mode?
  • User understanding: Can users predict what it will do?

Common Patterns

The Assessment Pattern

Before doing X, assess Y. This prevents jumping to conclusions.

Before explaining [topic], first assess what the user 
already knows about [topic]. Ask specific questions to 
identify gaps. Only then proceed with instruction.

The Loop Pattern

Repeat a process until a condition is met.

Run the following loop until the plan is solid:
  1. Present the current plan
  2. Identify the weakest assumption
  3. Challenge that assumption
  4. Update the plan
  5. Repeat from step 1

The Output Pattern

Structure results for easy consumption.

Format your output as:
  ## Summary
  [One paragraph]
  ## Key Points
  - [Point 1]
  - [Point 2]
  ## Next Steps
  1. [Step 1]
  2. [Step 2]

Skill Quality Indicators

A great skill is:

  • Predictable — users know what they'll get
  • Focused — does one thing well
  • Consistent — same input, same mode
  • Self-documenting — the instructions explain themselves
  • Composable — works well with other skills
The Composition Principle Skills should chain naturally. A skill that can't combine with others is limited. Design skills that work as building blocks.
