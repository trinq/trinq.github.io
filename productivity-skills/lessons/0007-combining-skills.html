Combining Skills

Chain skills together for powerful, intentional workflows

The power of skills isn't just individual capabilities — it's how they compose.

The Core Insight

Skills are building blocks. Individually useful, but when combined intentionally, they create workflows that are greater than the sum of their parts. The skill you use next shapes what you get from the one you use now.

The Rule Match the skill to the phase. Don't brainstorm when you should be grilling. Don't build when you should be learning. Know where you are in the process.

Step 1 /grill-me

Use for: Hard bugs. Diagnose systematically, validate understanding with grilling, then implement the fix.

How to Combine Skills

Sequential Chaining

Use one skill, then another in sequence:

/brainstorm ways to improve our onboarding flow
Brainstorming mode
[generates 15 ideas]
The personalized email sequence looks promising. /grill-me

[interviews about implementation, edge cases, success metrics]
Let's build it. /tdd

[writes tests first, then implements]

Use Case Examples

Building a New Feature

Scenario: "We need a dark mode toggle for our app."
/brainstorm implementation approaches /grill-me chosen approach /tdd build with tests /review verify implementation

Learning a New Framework

Scenario: "I want to learn Next.js and build a portfolio."
/teach Next.js fundamentals /brainstorm portfolio features /grill-me scope the MVP /build implement incrementally

Writing a Skill

Scenario: "I need a skill to automate my standups."
/brainstorm what it should do /grill-me refine the design /teach (optional) learn from existing skills /build write the skill

Fixing a Bug

Scenario: "Memory leak in production, need to diagnose and fix."
/systematic-debugging find root cause /grill-me validate understanding /tdd write regression test first /fix implement the solution

Anti-Patterns

✓ Do

  • Match skill to phase
  • End one skill before starting another
  • Use the result of one as input to next
  • Stay in the same mode until done

✗ Don't

  • Switch skills mid-phase
  • Evaluate while brainstorming
  • Build before understanding
  • Skip the grilling step

Choosing the Right Chain

The Decision Tree
  • Need ideas? → Start with /brainstorm
  • Have an idea? → Follow with /grill-me
  • Need to learn? → Start with /teach
  • Ready to build? → Use /tdd or /build
  • Something broken? → Start with /systematic-debugging

Pro Tips

Name your workflows. "The Creation Workflow" is memorable. "That thing where I brainstorm then grill then build" is not.
Default to the simplest chain first. /brainstorm → /grill-me → /build handles 80% of cases. Add more steps only when needed.
If stuck, go back a phase. If you're building and feel lost, you probably skipped grilling. Back up and validate.
Document your custom chains. Write them down somewhere accessible so you can repeat successful patterns.

Create Your Own Workflows

The real power: combine skills in ways that fit your process. The best chain is the one that works for you.
