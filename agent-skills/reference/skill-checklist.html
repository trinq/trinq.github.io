The Skill Checklist

A one-page audit rubric for agent skills · derived from Matt Pocock's "The Missing Manual"

Order matters. Audit Trigger first — every later axis can be wasted work if the skill never fires (or fires too often).

The four axes

AxisQuestionFailure mode it catches
Trigger Who fires this — me or the agent? What's the cost? Context bloat from accidental model-invocation; pilot fatigue from too many user-invoked skills
Structure Are steps and reference separated? Is branching material pushed behind context pointers? Bloated skill.md full of crud that applies to one branch only
Steering Do I use leading words? Are they consistent? Is each step getting enough leg work? The agent doing what I literally said, not what I meant
Pruning Is every paragraph a no-op survivor? Any sediment? Single source of truth? Bloated skills full of instructions that don't change behaviour

Trigger — quick audit

Frontmatter question:
  Has `disable-model-invocation: true`  → user-invoked
  Has a `description:`                    → model-invoked (description lives in agent context)

Decision tree:
  Will the agent reliably know when to fire it WITHOUT seeing the description?
    yes  → user-invoked (cheaper context)
    no   → model-invoked (cheaper cognitive load)
  Cost check:
    Every model-invoked skill adds N tokens to every request.
    Every user-invoked skill adds one thing the pilot must remember.
    Pick the cheaper cost for your situation.

Structure — quick audit

  • Two units only: steps (imperative) and reference (descriptive).
  • Is the skill.md minimal? Move branch-specific reference behind context pointers (link to another file, don't inline).
  • If material applies to only one branch of the skill's use cases, push it out.
  • Reference docs (glossary, templates, examples) get their own files and get linked.

Steering — quick audit

  • Pick 2–4 leading words — dense terms the agent will recognise from prior training (e.g. vertical slice, red-green-refactor, evidence before assertions).
  • Repeat each leading word throughout the skill so it shows up in the agent's reasoning traces.
  • If the agent isn't doing enough leg work on a step: split the skill. Hide the future goal so the agent can't see it.
  • Test: after running the skill, scan the agent's thinking trace for your leading words. If they're absent, the steering is too weak.

Pruning — quick audit

  • Run the deletion test on every paragraph: delete it, re-run the skill, compare output. If unchanged → no-op, delete permanently.
  • Hunt sediment: irrelevant legacy paragraphs. Most likely accumulated from copy-paste between skills.
  • Hunt crud: generic material that doesn't apply to every branch.
  • Enforce single source of truth: each piece of reference material (PRD template, glossary term, test seam definition) lives in one place.
The deletion test is the one technique that consistently catches no-ops. Build it into your workflow: when reviewing any skill (yours or community), delete paragraphs first and watch what changes.

Five-minute audit workflow

  1. Open SKILL.md. Read the frontmatter.
  2. Trigger — is this user or model invoked? Right for this skill?
  3. Structure — is anything branch-specific inline? Push it out.
  4. Steering — what are the leading words? Run the skill and check the trace.
  5. Pruning — deletion-test each paragraph. Kill no-ops.

Total budget: 5 minutes per skill. Most skills need at most 1–2 axis fixes.
