Agent Skills Glossary

The terms of art · cheat sheet for re-reading the lessons

Terms are listed in roughly the order they appear in the lessons. Definitions are minimal — go to the lessons for nuance and examples.

The framework

Skill

A markdown file (typically SKILL.md) with frontmatter that an agent loads to perform a specific task. The unit of authored behaviour.

Skill hell

Matt's term for the state of having many skills you can't tell good from bad, so they don't deliver the value they promise.

The four axes

Trigger · Structure · Steering · Pruning — the checklist for auditing a skill, in this order. From Matt Pocock, "Building Great Agent Skills: The Missing Manual".

Trigger

Trigger

How a skill gets loaded: by the user explicitly (e.g. /skill-name) or by the model deciding to based on the description.

User-invoked skill

Only the user can fire it. The agent cannot — the description either doesn't exist or is hidden (disable-model-invocation: true). Lower context load, higher cognitive load on the pilot.

Model-invoked skill

The agent can fire it itself based on the description sitting in its context. Higher context load, lower cognitive load on the pilot.

Context pointer

A short description (frontmatter description:) that sits in the agent's context and points to the full skill file. Lets the agent choose to load more.

Context load

The cost imposed on the agent by having skills loaded. Every model-invoked skill adds its description to every request.

Cognitive load

The cost imposed on the user by having to remember and decide which skill to fire.

Structure

Structure

The internal layout of a skill. Two units: steps (imperative actions) and reference (descriptive material). Branching content goes behind context pointers.

Step

An imperative action the agent takes inside a skill: "Read the file. Run the test. Ask the user."

Reference

Descriptive material the agent reads to inform its actions: a glossary, a template, a PRD example.

Crud

Generic material inside a skill that doesn't apply to every branch of its use cases. Should be pushed behind a context pointer.

Steering

Steering

How the skill gets the agent to do what you want. Primarily through leading words, secondarily by managing leg work via skill splitting.

Leading word

A term that packs dense meaning into a small phrase, repeated throughout a skill so the agent adopts it in its reasoning traces. Examples: vertical slice, red-green-refactor, evidence before assertions.

Vertical slice

Matt's canonical example leading word: ship a thin end-to-end slice (UI → API → DB) instead of a horizontal layer (all DB, then all API, then all UI). Used as the antidote to "code layer by layer".

Leg work

The depth of effort the agent puts into a single step. Plan-mode agents famously do too little ask clarifying questions leg work because they can see the future goal of "create plan".

Hidden future goal

The technique of splitting one skill into two so the agent working on step 1 cannot see step 2. Forces more leg work on step 1 because there is no tempting shortcut toward step 2.

Pruning

Pruning

The final pass to remove sediment, crud, and no-ops from a skill. Order matters: prune last, after Trigger / Structure / Steering are right.

Sediment

Irrelevant legacy material accumulated as multiple people contributed without deleting. Most common in skills authored collaboratively.

No-op

An instruction that looks like it does something but doesn't actually change the agent's behaviour. Caught by the deletion test.

Deletion test

Delete a paragraph from a skill. Re-run the skill. Compare the agent's output to before. If unchanged, the paragraph was a no-op — leave it deleted.

Single source of truth

Each piece of reference material (PRD template, glossary term, test seam definition) lives in exactly one place. Reference it from elsewhere rather than copying.

Harness terms

Frontmatter

The YAML block at the top of a SKILL.md containing name, description, disable-model-invocation, and other harness-specific fields.

Reasoning trace

The agent's "thinking" output before its reply. A leading word shows up here if the steering worked.
