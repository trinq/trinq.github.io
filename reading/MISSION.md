# Mission: Read nonfiction that actually sticks

## Why
You already finish 25+ nonfiction books a year, so the bottleneck isn't throughput — it's retention. The books blur together within weeks, and the ideas you paid attention and time to absorb never become part of how you think, decide, or act. You want reading to compound: each book should leave a residue that the next book builds on, and the things you read should start changing what you do this month.

## Success looks like
- You can state the central argument of any book you read in the last 6 months, in your own words, in under a minute.
- You can name the strongest objection to that argument — the one a hostile critic would raise — without re-reading the book.
- You carry a small, written set of "ideas I'm actively using" extracted from books, and at least one of them changes a real decision in your life each month.
- Reading time per book drops sharply (target: 30–60 minutes per serious nonfiction book) without any loss of retention or critical depth.

## Constraints
- You already have a NotebookLM account, so the tool is on the table — no need to teach a generic LLM workflow.
- You read for personal growth, not professionally. Lessons should optimise for use in your own life, not for citation in a report.
- You have less than an hour a day to spend on this practice. Lessons must be completable in 15–30 minutes.
- You learn best by doing, not by being lectured at. Every lesson should end with a concrete task on a real book.

## Out of scope
- Speed-reading techniques that trade depth for throughput (we are explicitly not doing that).
- Fiction reading. The workflow is designed for argument-driven nonfiction.
- Building a "second brain" / note-taking system (Obsidian, Roam, etc.). We will use the smallest possible written artifact — one page per book — and revisit tooling only if it becomes the bottleneck.
- Reading more books. The metric is retention and use, not volume.
