Glossary

Vocabulary for the Harden Windows Security on-prem series. One sentence per term, lesson links where the term is taught. Read once, refer back often.

Every term in the series vocabulary sits in one of three buckets: architecture (Windows / AD / kernel concepts), policy surface (the GPOs, ADMX settings, baselines, and rules), telemetry / IR (logs, event IDs, forensics habits). Tags in the glossary entries mark the bucket; lesson links in each entry point to the lesson that teaches the term in operational context.

How to use this card
During a real admin session — a Tuesday ticket, a staging review, an incident call — this is the lookup surface. Use the term-index (below) or Ctrl-F. Every entry links forward to the lesson where the term is developed; the lesson’s design-decisions record is the deeper rationale when the one-sentence definition isn't enough.

Architecture & concepts

A
AC (Active Directory) architecture
Microsoft’s on-prem directory service (LDAP / Kerberos / Group Policy); the single forest is the assumed scope for this series.
Taught in L3.
ACL (Access Control List) architecture
The discretionary access list on an AD object, a file, a registry key, or a share; “who can do what” enforcement at the boundary.
Referenced in L2 (boundary 5 misconfiguration path).
Administrator — the L1 anchor architecture
A user account with the local Administrators group membership on a Windows machine; there is no security boundary between this and the kernel, which collapses every layer below it (L1, L2, L5).
Anchored in L1; revisited in L7.
AppContainer architecture
A sandboxed process capability profile used by UWP / Store apps; L2 boundary row 6.
Defined in L2.
AppLocker architecture
Legacy user-mode allow-listing system; the modern replacement is WDAC user-mode policy (L6).
Referenced in L6 for trade-off comparison.
ASR rule (Attack Surface Reduction rule) policy surface
A behavior policy in Microsoft Defender that blocks specific process behaviors (Office spawning children, scripts launching executables, etc.) without needing a signature.
Anchored in L5; cross-referenced in L7 (BYOVD bypasses ASR).
B
Baseline (Microsoft Security Compliance Toolkit baseline) policy surface
The curated, downloadable, ADMX-backed set of GPO settings Microsoft publishes per Windows release; a starting point, not a checklist.
Anchored in L4.
BitLocker policy surface
Disk-volume encryption with TPM-anchored key sealing; mandatory for any hardened endpoint, covered by L4’s storage baseline category.
Listed in L4's five baseline categories; referenced in baselines cheatsheet.
Boundaries — the seven L2 boundaries architecture
CPU ring, process, user (Standard vs Admin), network, AD domain, AppContainer, VBS; the architecture rows where Windows enforces privilege separation.
Anchored in L2; mapped to enforcement layers in L5, L6, and L7.
BYOVD (Bring Your Own Vulnerable Driver) architecture
An attack class using a real, signed, kernel-mode driver with a publicly known vulnerability; the driver does exactly what it was signed to do — L5 and L6 do not stop it.
Anchored in L7.
C
Capcom.sys class architecture
The canonical BYOVD reference example — a signed “Capcom” test driver with a kill-any-process IOCTL.
Anchored in L7's BYOVD history table.
CIM (Code Integrity Module) architecture
The kernel-side module that consults WDAC policy and decides whether a binary loads; consults at every driver / executable load.
Referenced in L6; telemetry events live in the CodeIntegrity log.
Code Integrity / `CodeIntegrity/Operational` log telemetry / IR
The kernel-owned Event Log channel where WDAC and HVCI decisions are recorded; events 3076 / 3077 / 3099 / 3091 are the load-bearing telemetry.
Anchored in L6; revisited in L7's IR playbook.
Credential Guard policy surface
A VBS-isolated region that stores Kerberos TGTs and NTLM hashes out of reach from the running kernel; L2 boundary row 7 made practical.
Anchored in L7.
Cpu ring architecture
The hardware privilege separation between Ring 0 (kernel) and Ring 3 (user mode); L2 boundary row 1; the ring that WDAC and HVCI enforce.
Anchored in L2; configured by L6 and L7.
D
DC (Domain Controller) architecture
The Active Directory role that hosts the Kerberos KDC and the AD database; tier-0; never first in a kernel-mode rollout.
Rolled-out-last discipline in L6's breakages section.
DFCI (Device Firmware Configuration Interface) policy surface
UEFI-level settings management from Windows; out of scope for the on-prem junior-admin series.
Listed in MISSION.md as a deferred vocabulary term.
DRTM (Dynamic Root of Trust Measurement) architecture
Late-launch measured-boot root of trust; tier-4 posture, mentioned in L1's threat-actor table, out of scope for the on-prem curriculum.
Referenced in L1.
E
EDR (Endpoint Detection and Response) telemetry / IR
A class of security product (Defender for Endpoint, CrowdStrike, etc.) that aggregates process / network / file telemetry centrally; ASR can run without it.
Distinguished from ASR in L5's opening table.
Event ID 3076 (CodeIntegrity) telemetry / IR
A binary was allowed in Audit, or would have been blocked — the load-bearing WDAC telemetry event.
Anchored in L6.
Event ID 3077 (CodeIntegrity) telemetry / IR
A binary was actually blocked from launching (Enforced mode); the 3076 / 3077 ratio is the WDAC leading indicator.
Anchored in L6.
Event ID 3091 (CodeIntegrity) telemetry / IR
WDAC policy load summary — shows which base / supplemental policies applied at boot.
Anchored in L6.
Event ID 3099 (CodeIntegrity) telemetry / IR
An “allow” event was triggered — an allowed binary launched under the WDAC audit policy.
Anchored in L6.
Event ID 1121 (Defender ASR) telemetry / IR
An ASR rule fired in Audit mode; the rule “would have blocked but allowed.”
Anchored in L5.
Event ID 1122 (Defender ASR) telemetry / IR
An ASR rule fired in Block or Warn mode; the rule actually stopped something.
Anchored in L5.
ETW (Event Tracing for Windows) telemetry / IR
The kernel-side trace facility; tracelog -start with the CodeIntegrity provider enabled is the L7 IR playbook for kernel-depth forensics.
Anchored in L6 (mentioned) and L7 (operational).
F
Floor / Choice / Disable (L4 taxonomy) policy surface
The per-setting decision vocabulary: floor (apply by default), choice (adopt per environment context), disable (record and opt out).
Anchored in L4; applied per ASR rule in L5.
G
GPO (Group Policy Object) policy surface
The AD-authoritative policy container — configuration + security filtering + WMI filter, linked to OUs / sites / the domain root; the on-prem distribution mechanism.
Anchored in L3.
GPMC (Group Policy Management Console) policy surface
The MMC snap-in for authoring, linking, and modeling Group Policy; the on-prem admin’s main GPO interface.
Referenced throughout L3.
`gpupdate /force` policy surface
The client-side command that refreshes all applied GPOs immediately (skipping the 90-minute background cycle); use during staging, never as a habit.
Referenced in L3 and L4.
`gpresult /h` telemetry / IR
The verification command that produces an HTML report of which GPOs applied to a machine and in what order; the “did my GPO apply?” first-line tool.
Anchored in L3.
H
HotCakeX / Harden-Windows-Security architecture
The series’ anchor text — a curated public compilation of official-Microsoft Windows hardening methods; cloud-first framing, translated to on-prem by this series.
Cited throughout; full citation in RESOURCES.md.
HVCI (Hypervisor-protected Code Integrity) policy surface
Kernel-mode memory-integrity enforcement that runs inside the VBS hypervisor; blocks kernel code modification even from allowed identities; hardware prerequisites apply.
Anchored in L7.
I
IOMMU (Input-Output Memory Management Unit) architecture
The hardware virtualization layer (Intel VT-d / AMD-Vi) used by the VBS hypervisor for memory isolation between normal and VBS-isolated kernels; required for HVCI and Credential Guard.
Anchored in L7 hardware prerequisites section.
K
Kernel-mode driver architecture
A driver that runs in Ring 0 with direct hardware / kernel access; the BYOVD attack surface; configured by L6's WDAC kernel-mode and L7's HVCI.
Anchored in L6; threat-modeled in L7.
KRBTGT architecture
The AD account whose hash signs every Kerberos TGT; compromising it = compromising the entire domain (boundary 5 collapse).
Referenced in L2 and L3.
L
LAPS (Local Administrator Password Solution) policy surface
The Microsoft mechanism for rotating local Administrator passwords automatically per machine; breaks credential reuse at L2 boundary 4.
Referenced in L2 (boundary 3 floor decision) and L4 (account category).
LGPO.exe policy surface
Microsoft’s command-line tool for applying registry-based policy without AD — and, just as importantly, for verifying what policy a machine actually has applied.
Anchored in L4; reframed as a verification tool there.
LOLDrivers telemetry / IR
The public security-research catalog of disclosed-vulnerable Windows drivers; the substrate the Microsoft recommended driver block list is curated from.
Anchored in L7.
LSASS (Local Security Authority Subsystem Service) architecture
The Windows process that holds live logon credentials in memory; the credential-theft target for Mimikatz-class tools; isolated by Credential Guard.
Anchored in L2 and L7.
policy surface
LSDOU (Local / Site / Domain / OU)
The Group Policy processing order; Local policies first, then Site, then Domain, then OU (link-order ties at each level). Junior-admin mistake: assume lower-numbered = higher-priority.
Anchored in L3.
M
MitM (Man-in-the-Middle) architecture
An attack that sits between two trust endpoints; SMB-signing and Kerberos-authenticated RDP are the relevant network MitM mitigations in the series scope.
Touched in L2's boundary 4 row.
MOTW (Mark-of-the-Web) policy surface
The zone-identifier attached to files downloaded from the Internet; consumed by SmartScreen and a small subset of ASR rules.
Listed in MISSION.md as vocabulary deferred to glossary.
MSCT (Microsoft Security Compliance Toolkit) policy surface
The Microsoft-published toolkit containing the Windows / Office / Edge baselines (ADMX + GPO backup), Policy Analyzer, LGPO.exe, and Set-RegistryPolicy; the canonical artifact source.
Anchored in L4.
MSIX architecture
The modern Windows app packaging format with AppContainer support; L2 boundary row 6 surface.
Referenced in L2.
N
NTLM architecture
The legacy Windows authentication protocol; vulnerable to credential-theft and relay attacks; mitigations are blocking or restricting NTLM via GPO, and the Protected Users group.
Referenced in L2 and L7 (Credential Guard breakage).
O
OU (Organizational Unit) architecture
The AD container used to scope GPO application; the load-bearing GPO-placement primitive (link at the lowest OU that makes sense).
Anchored in L3.
P
PAW (Privileged Access Workstation) architecture
A workstation dedicated to tier-0 administration; tier-0 isolation is architectural, not GPO-configurable.
Referenced in L2's scenario A.
Policy Analyzer telemetry / IR
The MSCT tool that compares a machine’s effective policy against a baseline; the “did my GPO apply correctly across the fleet?” tool.
Anchored in L4.
PPL (Protected Process Light) architecture
A Windows process-protection designation that restricts which processes can open a handle to it; the protection LSASS uses when Credential Guard is enabled.
Listed in MISSION.md as deferred vocabulary term.
Protected Users policy surface
A built-in AD group whose members cannot use NTLM, DES, or Kerberos delegation; one of L2 boundary 5's GPO-configurable defenses.
Referenced in L2's scenario C.
R
Realtek audio driver class (BYOVD example) architecture
OEM-shipped audio drivers whose vulnerable IOCTLs are a published BYOVD example; demonstrates that drivers you have already deployed are the attack surface.
Anchored in L7's BYOVD history table.
Recommended Driver Block List (Microsoft) policy surface
The pre-built, signed, kernel-mode WDAC policy Microsoft ships and updates automatically; the floor decision for BYOVD-class defenses.
Anchored in L6 (as policy); revisited in L7 (as BYOVD mitigation).
S
Secure Launch / DRTM architecture
Hardware-rooted measured-boot (DRTM); tier-4 posture, out of scope for the on-prem junior-admin curriculum.
Referenced in L1's threat-actor table.
Security boundary (L2 definition) architecture
A privilege-separation line an attacker at a lower privilege level cannot cross without first breaking the line itself; the row concept in L2's table.
Anchored in L2.
SLAT (Second-Level Address Translation) architecture
The hardware-virtualization CPU feature used by the VBS hypervisor; required along with IOMMU for HVCI and Credential Guard.
Anchored in L7 hardware prerequisites section.
SMB signing policy surface
Cryptographic signing of SMB packets to prevent relay attacks; L2 boundary 4's GPO-configurable primary defense.
Referenced in L2's scenario B.
SmartScreen policy surface
Microsoft's URL / file reputation service; user-mode check (not a security boundary); covered by L4's app execution category.
Listed in L1's tier-1 defense.
SYSVOL architecture
The AD-distributed file share that hosts GPO ADMX templates and scripts; one of L3's failure modes (SYSVOL lag).
Anchored in L3.
T
Threat-actor hierarchy (tiers 1–4) architecture
Script kiddie (tier 1) → organized crime (tier 2) → hacktivist/insider (tier 3) → nation-state (tier 4); calibration target for hardening investments.
Anchored in L1.
policy surface
Tier-0 / Tier-1 / Tier-2 (AD admin model)
The privilege-tiering model for AD administration: tier-0 = DCs and identity management; tier-1 = servers; tier-2 = workstations.
Referenced in L2's scenario A and L3's OU pattern.
TPM 2.0 architecture
The hardware root-of-trust chip; required for Credential Guard; recommended for HVCI; anchors BitLocker key sealing.
Anchored in L7.
U
UAC (User Account Control) architecture
The Windows elevation-prompt mechanism; the L2 row-3 boundary's enforcement layer (when configured correctly).
Anchored in L2.
V
VBS (Virtualization-Based Security) architecture
The Windows feature that uses the Hyper-V hypervisor to isolate a kernel-region from the normal kernel; the substrate for Credential Guard and HVCI; L2 boundary row 7.
Anchored in L2 and L7.
W
WDAC (Windows Defender Application Control) policy surface
Policy-driven executable-identity enforcement; kernel-mode blocks unsigned / blocked drivers, user-mode blocks unallowed applications; complements ASR, does not replace it.
Anchored in L6.
WEF (Windows Event Forwarding) telemetry / IR
The Windows mechanism for forwarding event logs from source machines to a central collector; the minimum-viable on-prem path for ASR / WDAC telemetry without a SIEM.
Anchored in L5 and L6.
WMI filter policy surface
A WQL query attached to a GPO; the GPO applies only to machines whose WMI query returns true; an alternative to security filtering.
Anchored in L3.
