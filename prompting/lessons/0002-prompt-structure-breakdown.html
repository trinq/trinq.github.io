Lesson 0002: Prompt Structure Breakdown

System prompt vs user prompt — đi sâu vào từng thành phần

System Prompt vs User Prompt

Từ video, Anthropic phân chia rõ: system prompt chứa thông tin tĩnh, user prompt chứa dữ liệu động.

System prompt = stable, cached, không đổi giữa các lần gọi
User prompt = dynamic, thay đổi mỗi lần

10 Components Chi Tiết

1
Task Context
Both

Claude là ai? Đang làm gì? Mục đích cuối cùng là gì?

You are an AI assistant helping a human claims adjuster
reviewing car accident report forms in Swedish.
2
Tone Context
Both

Giọng văn, phong cách phản hồi. Quan trọng để tránh Claude "bịa đặt".

Stay factual and confident. If you cannot determine
what happened, say you cannot determine — do not guess.
3
Background / Data / Documents
System

Thông tin tĩnh, không thay đổi giữa các lần gọi. Đặt ở system prompt.

This is a Swedish car accident form. The form will be in Swedish.
It has a title, two columns (representing Vehicle A and B),
and 17 rows describing different aspects of the incident.
Tip: Đây là nơi dùng prompt caching. Nội dung này luôn giống nhau → cache được.
4
User Content
User

Dữ liệu động từ user — form thực tế cần analyze, ảnh, file.

Please analyze this accident form and determine who is at fault:

<form>
[the actual form content / image]
</form>

<sketch>
[the drawing of the accident]
</sketch>
5
Detailed Instructions
Both

Step-by-step cách Claude phải suy nghĩ. Order matters — thứ tự ảnh hưởng kết quả.

Step 1: Carefully examine the form first. Make a list of
       what boxes are checked for each vehicle.
Step 2: Then examine the sketch, keeping in mind your
       understanding from the form.
Step 3: Make your final assessment based on both sources.
Key insight: Như con người, Claude cần context (form) trước khi hiểu sketch. Đổi thứ tự = đổi kết quả.
6
Examples (Few-shot)
System

Input → Output mẫu để steer model vào đúng hướng. Dùng khi có edge cases khó.

<example>
  <input>
    [example form with clear fault determination]
  </input>
  <output>
    Based on the form, Vehicle A was marked [details].
    The sketch confirms [details].
    Verdict: Vehicle B is at fault.
  </output>
</example>
7
Conversational History
User

Cho multi-turn conversation. Không dùng trong demo nhưng quan trọng cho chatbot thực tế.

<history>
  User: [previous question]
  Claude: [previous response]
  User: [new question]
</history>
8
Reminders / Guidelines
Both

Nhắc lại điều quan trọng, tránh hallucination. Đặt cuối prompt.

Reminder:
- Only make claims if evidence exists in the provided documents
- If a checkbox is unclear, say "unclear" — do not assume
- If the sketch is unintelligible, say so
9
Output Formatting
Both

Định nghĩa format đầu ra. Quan trọng cho data engineering — bạn cần parse được output.

Format your response using these XML tags:
<form_analysis>...</form_analysis>
<sketch_analysis>...</sketch_analysis>
<verdict>...</verdict>
10
Pre-filled Response
System

Điền sẵn đầu output để shape kết quả. Claude sẽ tiếp tục từ đó.

When outputting your response, begin with:
<verdict>
Tip: Thay vì parse preamble thừa, chỉ cần parse phần bạn cần trong XML tags.

Exercise

Lấy một prompt bạn đang dùng (trong Copilot, Claude Code, hoặc Cursor). Phân tích nó thuộc components nào:

  • System prompt có bao gồm task context + tone?
  • User content có được wrap trong delimiters?
  • Có step-by-step instructions không?
  • Output format đã được định nghĩa chưa?
