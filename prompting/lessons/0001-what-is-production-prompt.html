Lesson 0001: What is a Production Prompt?

Khác gì chat? Tại sao cần cấu trúc?

Chat vs Production Prompt

Hầu hết chúng ta dùng LLM như chatbot — hỏi đáp, sửa code, brainstorm. Đó là conversational prompting.

Production prompt khác. Mục tiêu: đánh bại LLM 1 lần, lần nào cũng đúng.

"You send one single message to Claude and have it nail the task the first time around without needing to move back and forth."
— Hannah Moran, Anthropic Applied AI

10 thành phần của Production Prompt

Phần này từ video "Prompting 101" của Anthropic Applied AI team:

  1. Task context — Claude là ai? Đang làm gì? Role và mục đích
  2. Tone context — Giọng văn, phong cách phản hồi. VD: "factual, confident, don't guess"
  3. Background/Data — Thông tin tĩnh không đổi. Đặt vào system prompt
  4. User content — Dữ liệu động từ user. Input thực tế
  5. Detailed instructions — Step-by-step cách Claude phải suy nghĩ
  6. Examples (few-shot) — Input → Output mẫu để steer model
  7. Conversational history — Nếu có multi-turn (không dùng trong demo này)
  8. Reminders — Nhắc lại điều quan trọng, tránh hallucination
  9. Output formatting — Định nghĩa format đầu ra trước
  10. Pre-filled response — Điền sẵn đầu output để shape kết quả

Tại sao cần system prompt riêng, user prompt riêng?

Trong video, họ phân tách:

  • System prompt = thông tin tĩnh, không đổi giữa các lần gọi. VD: cấu trúc form, role definition, examples
  • User prompt = dữ liệu động, thay đổi mỗi lần. VD: form thực tế cần analyze
Tip từ video: Dùng XML tags (<form>, <analysis>) để tổ chức. Claude được fine-tune trên XML, hiểu rõ cấu trúc này.

Điều Claude cần để làm tốt

  • Structure — Claude loves organization. Không viết tự do
  • Order — Thứ tự thông tin quan trọng. Như con người, Claude cần đọc form trước, sketch sau
  • Confidence bounds — Nói Claude khi nào thì "không biết thì hỏi lại" thay vì đoán
  • Examples — Few-shot là cách nhanh nhất để steer Claude vào đúng hướng
