Lesson 0004: Examples & Few-shot

Cách dùng examples để steer Claude đúng hướng

Tại sao Examples mạnh?

Christian trong video: "Examples or few-shot is a mechanism that really is powerful in steering Claude."

Thay vì mô tả bằng words, bạn cho Claude thấy: input như thế này → output như thế kia.

Few-shot prompting = cho 2-3 examples để define pattern thay vì giải thích bằng text.

Khi nào cần Examples?

  • Edge cases — trường hợp khó, nằm ngoài common sense
  • Format cụ thể — bạn muốn output theo một style nhất định
  • Domain knowledge — Claude thiếu context về domain của bạn
  • Grading criteria — bạn có labeled data cho "đúng" và "sai"

Cách viết Examples

Template

<examples>
  <example>
    <input>
      [Mô tả tình huống đầu vào]
    </input>
    <output>
      [Kết quả mong muốn - phải là output HOÀN CHỈNH]
    </output>
  </example>
  <example>
    <input>
      [Tình huống tiếp theo]
    </input>
    <output>
      [Kết quả tương ứng]
    </output>
  </example>
</examples>

Example: Tránh Insurance Fraud

Example Phát hiện insurance claim đáng ngờ 
<example>
  <input>
    Form shows Vehicle A hit from behind at stoplight.
    Sketch shows Vehicle A swerved into oncoming traffic.
    Checkboxes contradict the sketch description.
  </input>
  <output>
    Verdict: SUSPICIOUS
    Reasons:
    - Sketch contradicts form description
    - Checkboxes do not match incident type
    Recommendation: Manual review required
  </output>
</example>

3 nguyên tắc cho Examples tốt

  1. Diverse — Cover cả positive và negative cases
  2. Accurate — Output phải đúng 100% (không có "approximately" hay "something like")
  3. Concise — Không dài quá, chỉ đủ để define pattern
Tip từ Anthropic: Bắt đầu với 1-2 examples. Thêm khi thấy Claude sai ở một pattern cụ thể.

Quiz

Bạn đang viết prompt để Claude review code. Khi nào nên dùng few-shot?

  • A) Khi code luôn đúng và bạn muốn Claude nói đẹp
  • B) Khi bạn có các examples về code "tốt" vs "xấu" để define standard
  • C) Khi bạn muốn Claude viết thêm test cases
  • D) Khi không biết nên viết gì

→ Đáp án: B. Few-shot dùng khi bạn có examples để define pattern — ở đây là "code tốt vs code xấu".

Exercise

Lấy một task bạn hay dùng Claude/Copilot để làm. Viết 1 example:

<example>
  <input>
    [Tình huống thực tế bạn hay gặp]
  </input>
  <output>
    [Output bạn mong đợi]
  </output>
</example>

Test: Example này có cover được edge case khó nhất bạn từng gặp không?
