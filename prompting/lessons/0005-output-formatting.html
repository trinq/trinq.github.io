Lesson 0005: Output Formatting

Định nghĩa output để parse được dễ dàng

Tại sao output format quan trọng?

Christian trong video: "At the end of the day, you want your piece of information to be stored in your SQL database... you want the nitty-gritty information for your application."

Không phải lúc nào Claude cũng trả lời theo format bạn cần. Output formatting giúp bạn parse được đúng dữ liệu.

4 Loại Output Format

1. XML Tags (Khuyên dùng)

Đặt output sections trong tags có ý nghĩa.

Format definition
Format your response using these XML tags:
<analysis>
  [Your detailed analysis]
</analysis>
<verdict>
  [Your final verdict - only this will be stored]
</verdict>

2. JSON

Cho structured data cần parse bằng code.

JSON format
Output ONLY valid JSON:
{
  "verdict": "at_fault_vehicle_b",
  "confidence": 0.85,
  "reasons": ["checkbox_12_marked", "sketch_confirms"],
  "requires_manual_review": false
}

3. Markdown Tables

Cho summary mà con người đọc.

Table format
| Checkbox | Vehicle A | Vehicle B | Interpretation |
|---------|-----------|-----------|----------------|
| 1       | ✓         |           | Driving forward |
| 12      |           | ✓         | Stopped        |

4. Pre-filled Response

Điền sẵn đầu output để shape kết quả.

Prefill technique
Begin your response with:
<verdict>
[Claude will continue from here]

Claude sẽ tiếp tục từ đó → không có preamble thừa.

Best Practices

  • Chỉ định rõ — "Output ONLY valid JSON" thay vì "format nicely"
  • Ignore preamble — Nói rõ bạn sẽ ignore gì
  • Key fields required — Đánh dấu fields bắt buộc
  • Type constraints — "confidence must be float 0-1"
From the video: Sau khi thêm output formatting, họ chạy prompt và chỉ parse phần <verdict> — tất cả preamble Claude thêm vào được ignore.
