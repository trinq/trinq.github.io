Lesson 0006: Iterative Prompt Development

Cách test và improve prompt qua nhiều vòng

Prompt Engineering là Empirical

Trong video, Hannah nói: "Prompt engineering is a very iterative empirical science."

Không có công thức đúng ngay. Bạn test, thấy sai, sửa, test lại.

Feedback loop: Run → Observe → Add to system prompt → Repeat

Evolution của Prompt trong Video

Đây là 5 versions họ tạo trong demo:

V1

Baseline (không context)

Claude guess: "skiing accident on Chappangan street"

V2

+ Task & Tone context

Claude hiểu: car accident, nhưng còn thiếu form structure

V3

+ Background/Data (form structure)

Claude hiểu form, đưa ra verdict, nhưng còn loose

V4

+ Step-by-step instructions

Claude làm theo process: form → sketch → verdict

V5

+ Output formatting

Clean, parseable output — chỉ lấy verdict

Cách Build Prompt Hiệu Quả

Bước 1: Start Simple

Bắt đầu với basic task description. Chạy. Xem sai ở đâu.

Bước 2: Thêm Context

Khi Claude sai, hỏi: "Nó thiếu context gì?" Thêm vào system prompt.

Bước 3: Thêm Guardrails

Khi Claude hallucinate hoặc đoán, thêm reminders: "If uncertain, say so."

Bước 4: Thêm Examples

Khi Claude sai ở một pattern cụ thể, thêm example cho pattern đó.

Bước 5: Define Output

Khi output không parse được, thêm format definition.

Extended Thinking như Tool

Christian đề cập: Claude 3.7 và 4 có extended thinking. Bạn có thể đọc scratchpad của Claude để hiểu nó suy nghĩ thế nào.

Use case: Đọc thinking transcript để phát hiện Claude đang suy luận sai ở step nào → thêm instruction cho step đó.
