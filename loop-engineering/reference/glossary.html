Glossary

All terms from Andrew Ng's letter, plus adjacent vocabulary from Boris Cherny, Peter Steinberger, and Addy Osmani. One-stop lookup.

The three loops

Loop 1 — Agentic coding loop
The coding agent writes code, tests it, iterates against a spec (and optional evals) until the build passes. Cadence: every few minutes. Andrew's example: his daughter's typing app, where the agent worked autonomously for about an hour, checking the browser multiple times. Source: Andrew Ng, The Batch Issue 359.
Loop 2 — Developer feedback loop
The developer examines the current build and steers the coding agent by updating the spec, vision, or feature set. Cadence: tens of minutes to hours. The hard part is translating context advantage into a form the agent can use. Source: Andrew Ng, The Batch Issue 359.
Loop 3 — External feedback loop
Friends, alpha testers, A/B tests, production users — anyone with context the developer lacks. Cadence: hours to weeks. Feeds back into Loop 2 (vision) → Loop 1 (spec → agent). The only loop that cannot be fully automated. Source: Andrew Ng, The Batch Issue 359.

Andrew's vocabulary

Spec (product specification)
The behavioural, testable description of what the software should do. The input to Loop 1. Andrew: "it is still a lot of work to translate that vision into a specification for a coding agent to implement."
Evals
A dataset the agent can be scored against. Andrew: "a dataset against which to measure performance." Add evals after the system repeatedly runs into the same problem — not preemptively.
Context advantage
Andrew's preferred term over "taste" for what humans contribute. The gap between what the human knows and what the AI knows. Andrew: "since that gives us a clearer path to helping AI systems get better."
AI-native teams
Teams that use AI to shape product direction — gathering usage data, summarising customer feedback, competitive analysis. Term from Andrew's letter, Jun 2026.
Partial PM role
Engineers growing into product management because coding agents handle the typing. Andrew: "the hardest part is shaping the product vision."

The phrase "loop engineering"

Loop engineering
The practice of designing systems that prompt, verify, retry, and stop AI agents — instead of manually prompting them turn by turn. The phrase went viral in June 2026 after two tweets and one essay: Peter Steinberger's "designing loops that prompt your agents", Boris Cherny's "my job is to write loops", and Addy Osmani's Loop Engineering essay.
Loop (Addy Osmani's definition)
"A recursive goal where you define a purpose and the AI iterates until complete." Five building blocks: harness, loop contract, state layer, checker, and human checkpoint. Source: Addy Osmani, Elevate.
Boris Cherny quote
"I don't prompt Claude anymore. I have loops that are running. They're the ones that are prompting Claude and figuring out what to do. My job is to write loops." From AI Ascent 2026 with Sequoia's Lauren Reeder. Cherny is head of Claude Code at Anthropic.
Peter Steinberger quote
"You shouldn't be prompting coding agents anymore. You should be designing loops that prompt your agents." From @steipete on X, posted monthly as a reminder. Steinberger is the creator of OpenClaw.
OpenClaw
Peter Steinberger's framework. Referenced by Andrew Ng in the same sentence as Claude Code, positioning it as a parallel tool in the loop-engineering landscape.

Implementation primitives

/loop (Claude Code)
A Claude Code built-in command that runs a prompt on a schedule while the session is open. "All it is is you have Claude use cron to schedule a job for some point in the future, and it's a repeat job." — Boris Cherny, AI Ascent 2026. See official docs.
loop.md
A file that replaces the built-in /loop prompt with your own default instructions. Single default prompt, not a list of separate tasks. See Claude Code scheduled-tasks docs.
VISION.md
Steinberger's anchor file at the project root. Outlines what the product is, who it's for, what it's not. The agent reads it on every Loop 1 run. From explainx.ai coverage of Steinberger.
Loop contract
Addy Osmani's term for the observable contract that defines "done" and "failure" for a loop. Includes iteration caps, dollar budgets, and exit conditions. Source: Verdent Guides — How to Build a Coding Agent Loop That Stops Safely.
Independent verifier (or "checker")
The thing that says "yes, the agent's output is acceptable" — separate from the agent itself. Often a test suite, a type checker, a browser screenshot diff, or an LLM-as-judge. Andrew echoes this: "coding agents much more able to test their own code".

Why "context advantage" beats "taste"

FramingImplication
"Taste" Some things just feel right. You can't teach it. The AI either has it or doesn't.
"Context advantage" The human knows things the AI doesn't. Encode that knowledge into the spec, the eval, or the harness — and the gap shrinks.
