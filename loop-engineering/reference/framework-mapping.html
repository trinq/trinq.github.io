Framework Mapping

How Andrew Ng's 3 loops, LangChain's 4-Loop Ladder, and Matt Pocock's Agentic Engineering skills fit together. Print this and pin it above your monitor.

The vertical stack

Read top to bottom. Andrew Ng's 3 loops sit at the top (product cadence). LangChain's 4-Loop Ladder lives in the middle (agent infrastructure). Matt Pocock's harness lives at the bottom (dev ergonomics). Each layer composes with the layers below it.

Layer Framework What it answers
Product cadence Andrew Ng's 3 loops (this course) What runs at what speed? Who's in the loop? How fast does each cycle close?
Agent infrastructure LangChain's 4-Loop Ladder How does the agent run? How is it graded? When does it fire? How does it improve?
Dev harness Matt Pocock's Agentic Engineering What goes in CLAUDE.md? Which skills get installed? How do I queue work?
Agent loop primitives Loop Engineer (Vietnamese) / Agent Loop Skills How do I build a loop from scratch? What's the propose → run → score → keep/revert cycle?

Andrew's 3 loops ↔ LangChain's 4-Loop Ladder

LangChain's Ladder describes the agent's infrastructure. Andrew's loops describe the product's cadence. Where they overlap:

Andrew Ng (this course) LangChain (4-Loop Ladder) Relationship
Loop 1 — Agentic coding L1: Agent + L2: Verification + L4: Hill-climbing (partial) The LangChain agent, its rubric, and its self-improvement are all inside one Loop 1 cycle. Every time Loop 1 fires, it runs the full Ladder stack.
Loop 2 — Developer feedback L4: Hill-climbing (partial) + L3: Event-driven (partial) The developer's spec edits feed into L4 (which patches the agent). A cron-fired analysis agent can also live in L3.
Loop 3 — External feedback L3: Event-driven (mostly) Production data, alpha-tester feedback, A/B tests all fire L3 triggers that update vision → spec → agent.

The non-obvious mapping: Andrew's Loop 1 maps to LangChain's Loops 1+2+4 (the entire agent stack) because each Loop 1 iteration is an agent run that includes verification and possibly hill-climbing. Andrew's Loop 2 maps to LangChain's Loop 4 (the agent-improving-agent) because the developer's feedback becomes patches.

The clock / engine / transmission analogy

From Lesson 0001, expanded:

Metaphor Framework What you tune here
The clock Andrew Ng's 3 loops The cadence of your product. Minutes, hours, weeks. Who runs each tick.
The engine LangChain 4-Loop Ladder The agent's primitives. Tools, rubric, triggers, hill-climbing.
The transmission Matt Pocock's harness The dev ergonomics. CLAUDE.md, skills, queue, procedures.
The fuel The model Claude Code, Cursor, etc. The least interesting dial in 2026.

The mistake solo devs make: tune the engine (better prompts, better rubrics) without knowing the clock speed (which loop you're bottlenecked on). Result: ship fast in the wrong direction. Andrew's framework forces you to pick the clock speed first, then tune everything else.

The compounding view

Loop 4 of the 4-Loop Ladder is hill-climbing. If your agent improves by 1% every day, in a year it's 37.78× better. Where does that improvement come from in Andrew's framework?

  • Loop 1 evals accumulate — every recurring failure becomes a check, the rubric gets tighter.
  • Loop 2 specs tighten — every piece of context you encode into the spec is context you don't have to paste into chat.
  • Loop 3 vision sharpens — every cycle of external feedback prunes features that didn't matter and amplifies ones that did.

After 365 days of this, your agent is doing in 5 minutes what it used to take an hour of babysitting to produce. That's not because the model changed. It's because the harness — at all three layers — got tighter.

Final framing: Andrew Ng's 3 loops are the calendar. The 4-Loop Ladder is the engine. Matt Pocock's harness is the transmission. The compounding comes from running all three at once, day after day, and watching your agent do more of what you actually wanted.
