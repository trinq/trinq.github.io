Symptoms

You're stuck if…You're in…Fix
Watching the agent run for 20+ minutes, pasting context when it gets stuck Loop 2 (in disguise) Move the chat-injected context into the spec, an eval, or CLAUDE.md
Spec keeps growing without shipping Loop 2 (drift) Cap each spec at one Loop 1 iteration (~30 min of agent work)
"Decide what to change" takes days Loop 2 (clarity) Run grill-me until you can state the change in one sentence
No users, no feedback channel, no weekly ritual Loop 3 (missing) Pick 3 humans, create VISION.md, schedule a weekly 60-min ritual
Agent finishes, output is wrong, you ship anyway Loop 1 + 3 (broken) Add an eval. Stop shipping without one.

The "context advantage" loop

  1. You give the agent feedback based on context it doesn't have.
  2. Move that context into the system: a spec line, an eval, a CLAUDE.md entry, a VISION.md update.
  3. Next Loop 1 run reads the updated system. Doesn't need the chat.

Your job is to shrink your context advantage loop after loop. If the gap doesn't shrink, you're not actually using Andrew's framework.

The minimum viable loops

LoopMinimum viable
Loop 1 A spec, a test suite, a Claude Code session.
Loop 2 You, opening the build, deciding what to change, editing the spec. ≤45 min per round.
Loop 3 3 humans, a feedback channel, a weekly 60-min ritual, a VISION.md file.

Andrew's three lines to remember

"Given a product specification and optionally a set of evals, we can have an AI agent write code, test its work, and keep iterating until the code is bug-free and meets its specification."

— on Loop 1

"So long as the human knows something the AI does not, human-in-the-loop is needed to inject that knowledge into the system."

— on Loop 2

"This data informs the developer vision, which in turn continues to drive the detailed product spec, which in turn drives the coding agent."

— on Loop 3 → 2 → 1
